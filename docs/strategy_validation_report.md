## 용어 정의

- **5P (이전 버전)**: ma_window, buy_buffer, sell_buffer, hold_days, recent_months — recent_months 파라미터가 존재하던 시기의 파라미터 세트. recent_months는 최근 N개월의 청산 횟수에 따라 buy_buffer와 hold_days를 동적으로 조정하는 메커니즘이었다.
- **4P (현재 확정)**: ma_window, buy_buffer, sell_buffer, hold_days — recent_months 제거 후 확정된 파라미터 세트. 본 보고서의 모든 최종 수치는 4P 기준이다.
- **3P (교차 검증 전용)**: ma_window, buy_buffer, sell_buffer — hold_days도 제거(=0)한 최소 파라미터. §14~15의 교차 자산 검증에서만 사용. hold_days 제거의 부작용이 확인되어 최종 채택되지 않았다.

## 1. 분석 배경

### 1.1 분석 대상

QQQ 데이터를 시그널 소스로 사용하는 이동평균 밴드 기반 추세추종 전략 3종에 대해 과최적화 여부를 진단한다. 전략은 다음 순서로 진화하였다.

```
버퍼존 (QQQ) → 버퍼존 (TQQQ) → 버퍼존 ATR (TQQQ) + WFO
```

### 1.2 분석에 사용된 학술 프레임워크

본 분석은 다음 학술 연구에 기반한다.

- **Bailey, Borwein, López de Prado & Zhu (2014)** — "Pseudo-Mathematics and Financial Charlatanism": 백테스트 과최적화의 수학적 필연성 증명
- **Bailey & López de Prado (2014)** — "The Deflated Sharpe Ratio": 다중 검정과 비정규 분포를 보정하는 샤프 비율 검정
- **Bailey, Borwein, López de Prado & Zhu (2016)** — "The Probability of Backtest Overfitting": PBO(백테스트 과최적화 확률) 지표와 CSCV(조합적 대칭 교차검증) 도입
- **Harvey, Liu & Zhu (2016)** — "…and the Cross-Section of Expected Returns": 316개 팩터에 대한 다중 검정 분석
- **Harvey & Liu (2015)** — "Backtesting": 헤어컷 샤프 비율과 비선형 다중 검정 보정

### 1.3 과최적화의 핵심 원리

기대 샤프 비율이 0인(예측력 없는) 전략이라도, N개의 독립 시행 중 최선을 선택하면 운으로 달성 가능한 기대 최대 샤프 비율은 다음과 같다.

```
E[max SR] ≈ √(2 × ln(N))
```

| 시행 횟수(N) | 운으로 가능한 최대 SR | 해석                        |
| ------------ | --------------------- | --------------------------- |
| 27           | ~2.6                  | 파라미터 3개 단순 조합      |
| 432          | ~3.5                  | 버퍼존 QQQ/TQQQ 그리드 서치 |
| 1,728        | ~3.9                  | ATR WFO 그리드 서치         |

관측된 샤프 비율이 이 "운의 벽"보다 낮다면, 해당 전략의 성과가 순전히 운에 의한 것일 가능성을 배제할 수 없다.

## 2. 최종 결론 요약

> 이 절은 보고서 전체의 핵심 결론을 요약한다. 상세 근거는 §3~§24에 있다.

### 2.1 최종 파라미터 — 4개를 사용해도 되는 이유

| 파라미터        | 값        | 방어 등급 | 간략 근거                                                                                                                            |
| --------------- | --------- | --------- | ------------------------------------------------------------------------------------------------------------------------------------ |
| **ma_window**   | 200       | **A**     | 가장 널리 사용되는 장기 추세 필터. 자기실현적 예언 효과. 고원 확인됨: 150~300 어디를 선택해도 유사 (§18)                             |
| **buy_buffer**  | 0.03 (3%) | **A**     | 고원 중앙에 위치 (0.01~0.05). 0.07부터 성과 급락. 가장 깔끔한 고원 형태 (§18)                                                        |
| **sell_buffer** | 0.05 (5%) | **B+**    | 고원 내 위치 (0.03~0.07). "넓은 버퍼가 조기 청산 방지"라는 경제적 논거와 일치. 거래 수 함정 발견으로 0.10 이상은 통계적 무의미 (§18) |
| **hold_days**   | 3         | **A**     | 4가지 학술적 메커니즘으로 존재 정당성 S등급. 7자산×8값 고원 분석에서 교집합(3~5)의 좌단. 봉우리 아닌 고원의 일부 (§16~17)            |

각 파라미터 모두 "이 특정 값이 최적이라서" 선택한 것이 아니라 "고원 안의 합리적 지점"으로 선택했다. 어떤 파라미터든 고원 내 다른 값으로 바꿔도(예: MA=150, buy=0.02, sell=0.07, hold=5) 성과가 비슷할 것으로 예상된다.

> **이전 5P 체계에 대한 기록:** 이전에는 recent_months(최근 N개월 청산 횟수 기반 동적 조정) 파라미터가 존재했다. 전체기간 그리드 서치에서 recent_months=0이 최적이었으나, WFO 윈도우별 서치에서는 recent_months>0이 다수 선택되어 IS 과최적화를 유발하고 있었음이 밝혀졌다. 이에 따라 코드에서 완전 삭제되었다 (§20 참조).

### 2.2 과최적화가 아닌 이유

**검증 1 — 파라미터 안정성 (§13):** 432개 전 조합에서 Calmar > 0 (100%). 특정 파라미터 하나에 의존하는 봉우리가 아님.

**검증 2 — 교차 자산 범용성 (§14~15):** QQQ에서 찾은 파라미터를 고정하여 6개 다른 자산에 적용. 6개 전부 CAGR > 0%. SPY, EFA, GLD에서 양호한 Calmar. 특히 GLD(QQQ와 상관관계 ~0.05)에서의 성과는 "과거 데이터 암기"가 아닌 범용적 추세추종의 증거.

**검증 3 — 고원 확인 (§17~18):** 4개 핵심 파라미터 전부에 대해 고원 형태를 확인 (hold_days 56회 + 나머지 126회 = 182회 백테스트). 모든 파라미터의 현재 값이 고원 안에 위치. 봉우리(과최적화의 전형적 증거)가 발견되지 않음.

**검증 4 — 경제적 논거:** 전략이 특정 숫자가 아닌 시장의 구조적 특성(모멘텀 프리미엄, 단기 반전, 가짜 돌파)에 의존. 파라미터 각각에 대해 "왜 이 값이 작동하는가"를 학술 문헌으로 설명 가능.

### 2.5 WFO 검증 결과 요약 (엔진 버그 수정 후 최종)

> **중요:** 아래 수치는 2026-04-01 엔진 버그 수정(EMA 연속성 보장, Calmar 계산 통일) 이후의 최종값이다. 이전 보고서(2026-03-15)의 WFE Robust 2.09/2.38/2.39는 OOS EMA 리셋 버그에 의해 과대평가된 허상이었음이 확인되었다. 상세 원인은 §20.8 참조.

| 지표              | QQQ Dynamic | QQQ Fixed | TQQQ Dynamic | TQQQ Fixed |
| ----------------- | ----------- | --------- | ------------ | ---------- |
| Stitched CAGR     | 11.33%      | 11.97%    | 24.84%       | 26.27%     |
| Stitched MDD      | -35.64%     | -27.42%   | -71.57%      | -62.20%    |
| Stitched Calmar   | 0.318       | 0.436     | 0.347        | 0.423      |
| WFE Robust        | **0.93**    | **0.93**  | **0.19**     | **0.04**   |
| gap_calmar_median | -0.016      | —         | -0.088       | —          |
| PC 최대           | 0.447       | 0.441     | 0.691        | 0.684      |

**핵심 발견 1 — Dynamic ≈ Fixed 관계는 불변:** QQQ Dynamic 11.33% vs Fixed 11.97%, TQQQ Dynamic 24.84% vs Fixed 26.27%로 Fixed가 오히려 약간 우위. 파라미터를 매번 바꾸든 처음 것을 고정하든 결과가 거의 같다. **4P 동결 원칙의 근거는 유효하다.**

**핵심 발견 2 — WFO는 보조 검증 근거로 격하:** QQQ WFE 0.93은 "IS 성과의 93%가 OOS에서 재현"으로 양호하지만, 이전의 2.09처럼 "강한 방어 논거"로 사용할 수준은 아니다. WFO는 전략 유효성의 보조 확인 수단이며, 핵심 방어는 고원 분석(§17~18)과 교차 자산 검증(§15)이 담당한다.

**핵심 발견 3 — TQQQ의 실전 기대치 하향 필요:** TQQQ WFE 0.19는 "IS 성과의 19%만 OOS에서 재현"으로, 레버리지 상품의 본질적 한계(whipsaw 증폭)를 반영한다. TQQQ 단일 전체기간 백테스트 성과(CAGR 19.18%)가 실전에서 그대로 나올 것이라는 기대를 낮춰야 한다.

```
  - TQQQ는 실전 기대치를 낮춘 상태로 별도 모니터링
```

## 3. 전략별 과최적화 진단

### 3.1 버퍼존 (QQQ) — 위험도: 중간

**긍정적 요소:**

- 그리드 상위 4개의 Calmar(0.292~0.301)가 매우 안정적이며, "봉우리"가 아닌 "고원"의 신호

**우려 요소:**

- DSR 0.78은 95% 신뢰 수준(0.95) 미달

### 3.2 버퍼존 (TQQQ) — 위험도: 중간~높음

**진단:**

QQQ와 로직이 동일하고 매매 대상만 다르므로, 과최적화 진단의 핵심은 QQQ 전략에 있다. 다만 TQQQ 고유의 추가 위험이 존재한다.

### 3.3 버퍼존 ATR (TQQQ) + WFO — 위험도: 높음 → 전략 폐기

- OOS CAGR 표준편차(30.07%)가 평균(16.69%)의 약 2배로 성과가 극단적으로 불안정

## 4. CSCV/PBO 분석 해석

### 4.1 PBO의 의미

PBO(Probability of Backtest Overfitting)는 "IS에서 최적으로 선택한 전략이 OOS에서 중앙값 이하 성과를 보일 확률"이다. CSCV는 데이터를 6개 블록으로 나누어 C(6,3)=20가지 방법으로 IS/OOS를 분할하고, 각 분할에서 IS 최적의 OOS 상대 순위를 측정한다.

### 4.2 전략별 PBO 해석

| 전략           | PBO  | 해석                                                        |
| -------------- | ---- | ----------------------------------------------------------- |
| 버퍼존 (QQQ)   | 0.40 | 20회 중 8회 실패. 학술적 경고 수준(0.50) 미만이나 안심 불가 |
| 버퍼존 (TQQQ)  | 0.45 | 20회 중 9회 실패. 동전 던지기에 근접. 최적화의 가치 의심    |
| ATR WFO (TQQQ) | 0.65 | 20회 중 13회 실패. 명확한 과최적화. 최적화가 역효과         |

> PBO/DSR 분석은 이전 5P 체계에서 수행되었으며, recent_months 제거 후 재실행하지 않았다. 이유: ①CSCV/PBO 코드가 결론 완료 후 폐기됨 ②4P에서 파라미터 수가 줄어 PBO가 개선될 것으로 예상됨 ③교차 자산 검증(§15)과 고원 분석(§17~18)이 PBO보다 직접적인 과최적화 진단을 제공함.

### 4.3 PBO 해석 시 주의사항

PBO는 "모든 구간에서 성과가 좋아야 한다"는 기준이 아니다. PBO가 측정하는 것은 "IS 최적이 OOS에서 상대적으로 어디에 위치하는가"이다.

시장이 나빠서 모든 조합이 다 같이 손실을 보는 구간에서, IS 최적도 손실을 본다면, 이것은 PBO에 큰 영향을 주지 않는다 (IS 최적의 상대 순위가 중앙 근처이므로). PBO가 높아지는 것은 "IS 최적이 OOS에서 유독 나쁜" 경우에만 해당한다.

## 5. DSR 분석 해석

### 5.1 DSR의 의미

DSR(Deflated Sharpe Ratio)은 두 가지 왜곡을 보정한 샤프 비율 검정이다.

1. **다중 검정에 의한 선택 편향**: N개 조합 중 최선을 선택하면 SR이 부풀려짐
2. **비정규 분포**: 왜도와 첨도를 반영하여 SR의 통계적 유의성을 재계산

DSR > 0.95이면 95% 신뢰 수준에서 전략의 SR이 다중 검정 보정 후에도 유의미하다고 판단한다.

### 5.2 전략별 DSR 해석

| 전략           | DSR  | SR 관측 | SR 벤치마크 | Z-Score    | 해석                      |
| -------------- | ---- | ------- | ----------- | ---------- | ------------------------- |
| 버퍼존 (QQQ)   | 0.78 | 0.874   | 0.694       | +0.771     | 95% 미달이나 방향은 양(+) |
| 버퍼존 (TQQQ)  | 0.65 | 0.786   | 0.693       | +0.395     | 95% 미달, 약한 양(+)      |
| ATR WFO (TQQQ) | 0.35 | 0.707   | 0.800       | **-0.390** | **관측값이 운의 벽 미달** |

ATR WFO의 Z-Score가 음수라는 것은, 1,728개 조합을 탐색한 결과의 최고 SR이 순전히 운으로 달성 가능한 기대 최대 SR보다 낮다는 의미다. 이것은 가장 강한 과최적화 경고 신호이다.

### 5.3 DSR 계산 시 "진짜 N" 문제

DSR의 핵심 입력값은 "시도한 전략의 수(N)"이다. 그러나 실질적 N은 형식적 그리드 서치 조합 수보다 크다.

```
실질적 시행 횟수 추정:
  EMA vs SMA 선택                    → +2
  지표 유형 탐색 (밴드 구조 등)        → +α
  버퍼존 QQQ 그리드 서치               → +432
  TQQQ로 매매 대상 변경               → +432
  ATR 추가                           → +1,728
  WFO 설정 탐색                       → +추가 시행
  파라미터 단순화 탐색                 → +27
  ────────────────────────────────────
  보수적 추정: N ≈ 2,500~3,000+
```

DSR을 정직하게 계산하려면, 전략 개발 과정에서의 모든 암묵적 시행을 포함해야 한다. N=432가 아닌 N=2,500+으로 재계산하면 벤치마크 SR이 상승하여 DSR은 더 낮아진다.

## 6. WFO 성과 분석과 해석 원칙

### 6.1 WFO의 궁극적 목표

WFO(Walk-Forward Optimization)는 "모든 구간에서 돈을 벌어야 한다"는 도구가 아니다. WFO의 목표는 **"이 전략이 미래에 돈을 벌 가능성이 있는가"를 판단하는 진단 도구**이다.

추세추종 전략은 구조적으로 횡보장에서 손실을 보므로, 일부 OOS 윈도우의 손실 자체는 과최적화의 증거가 아니다.

### 6.2 WFO 윈도우 성과의 올바른 해석법

성과가 나빴던 OOS 윈도우를 분석할 때 구분해야 하는 두 가지 시나리오가 있다.

**시나리오 A — 시장이 나빠서 전체 조합이 다 안 좋은 경우:**

해당 OOS 기간에 전체 432개(또는 1,728개) 조합의 수익률 분포를 확인했을 때, 대부분이 음(-)이고 IS 최적 조합이 그 분포의 중앙 근처에 위치한다면, 이것은 **시장 환경의 문제**이지 과최적화가 아니다.

**시나리오 B — IS 최적이 유독 안 좋은 경우:**

전체 조합 분포에서 양(+)인 조합이 존재하는데 IS 최적만 하위권에 위치한다면, 이것은 **과최적화의 증거**이다. IS에서의 최적화가 과거 노이즈에 맞춰진 것이다.

## 8. 경제적 논거 평가

### 8.1 전략의 경제적 기반

버퍼존 전략의 핵심 논리: "QQQ가 장기 이동평균(200일 EMA) 위로 확실히 올라서면 추세가 시작된 것으로 판단하여 진입하고, 아래로 확실히 내려가면 추세가 끝난 것으로 판단하여 청산한다."

이것은 **모멘텀/추세추종**의 고전적 구현이며, 다음 학술 연구에 의해 지지된다.

- **Jegadeesh & Titman (1993, Journal of Finance)**: 3~12개월 모멘텀의 초과 수익을 최초로 문서화
- **Asness, Moskowitz & Pedersen (2013)**: 주식, 채권, 통화, 원자재 8개 시장에서 모멘텀의 범용성 확인
- **Moskowitz, Ooi & Pedersen (2012, Journal of Financial Economics)**: 58개 선물 시장에서 시계열 모멘텀(과거 수익률 부호 기반)의 유의미한 수익 보고
- **행동경제학적 기반**: 투자자의 과소반응(underreaction)과 과잉반응(overreaction)에 의한 추세 형성. 인간 심리에 기반하므로 구조적으로 지속될 가능성이 높음

### 8.2 경제적 논거의 방어력과 한계

경제적 논거는 전략의 **방향(direction)**을 지지하지만, 특정 **좌표(coordinates)**를 지지하지는 않는다.

| 주장                        | 방어 등급  | 근거                                                                                                |
| --------------------------- | ---------- | --------------------------------------------------------------------------------------------------- |
| "추세추종이 작동한다"       | **S등급**  | 수십 년간 다수 시장에서 재현된 학술적 증거                                                          |
| "200일 이동평균을 사용한다" | **A등급**  | 자기실현적 예언 + 고원 확인(150~300). §18 참조                                                      |
| "매수버퍼=3%가 최적이다"    | **A등급**  | 고원 중앙(0.01~0.05). 가장 깔끔한 고원 형태. §18 참조                                               |
| "매도버퍼=5%가 최적이다"    | **B+등급** | 고원 안(0.03~0.07). 경제적 논거 일치. 다만 거래 수 함정으로 0.10 이상 무의미. §18 참조              |
| "hold_days=3이 최적이다"    | **A등급**  | 학술 범위(2~5일)와 그리드 결과 수렴. 56회 고원 분석에서 3개 자산 고원 교집합(3~5) 확인. §16~17 참조 |

### 8.3 "시장 참여자들도 동일하게 생각한다"의 양면성

200일 이동평균은 실제로 가장 널리 사용되는 기술적 지표 중 하나이다. 이것은 두 가지 상반된 함의가 있다.

**긍정적 측면 — 자기실현적 예언(self-fulfilling prophecy):** 많은 트레이더가 200일 MA를 보고 매매 결정을 내리므로, 가격이 실제로 200일 MA 근처에서 반응하는 경향이 있다. 이는 전략의 경제적 기반을 강화한다.

**부정적 측면 — 혼잡 거래(crowded trade):** 동일한 신호를 보는 참여자가 많으면, 진입/청산 시점이 집중되어 슬리피지가 증가하고, 급격한 반전 시 동시 청산으로 손실이 증폭될 수 있다.

### 8.4 결론

경제적 논거만으로 과최적화를 면제받을 수는 없지만, **과최적화의 확률을 상당히 낮춰준다.** 특히 전략이 특정 파라미터 값이 아닌 시장의 구조적 특성(모멘텀 프리미엄)에 의존하는 경우, 파라미터 오염의 영향이 상대적으로 줄어든다.

## 9. 긍정적 요소

과최적화 우려에도 불구하고, 다음 요소들은 전략의 잠재적 유효성을 지지한다.

1. **경제적 논거가 존재**: 무작위 데이터 마이닝이 아닌 합리적 가설(추세추종)에서 출발한 전략
2. **상위 파라미터의 안정성**: QQQ 그리드 상위 4개의 Calmar 범위가 0.292~0.301로 매우 좁음
3. **슬리피지 반영**: ±0.3%의 거래 비용을 적용하여 비현실적 가정을 배제
4. **미래 참조 방지**: 다음 영업일 시가 체결로 look-ahead bias를 구조적으로 차단
5. **벤치마크 대비 우위**: 버퍼존 QQQ의 Calmar(0.30)는 B&H QQQ(0.12)의 2.5배
6. **1위 파라미터가 가장 단순**: 동적 조정 미사용은 복잡성 최소화의 신호

## 10. "이미 답을 본" 오염 문제

### 10.1 문제의 본질

432개 그리드 서치를 실행한 순간, 그 데이터셋에 대한 "순수한 OOS 검증"은 영원히 불가능해졌다. 파라미터를 3개로 줄여 27개 조합만 탐색하더라도, 연구자의 뇌에는 432개 탐색의 결과가 내재되어 있다. 이것은 학술적으로 **"연구자 자유도에 의한 데이터 오염(researcher degrees of freedom)"** 또는 **"암묵적 다중 검정(implicit multiple testing)"**이라고 한다.

### 10.2 해결 접근법

데이터 오염을 완전히 제거할 수는 없지만, 영향을 측정하고 최소화하는 방법이 있다.

**1층 — 연구자가 본 적 없는 데이터로 검증:**

- 새로운 시간 데이터: 파라미터를 동결하고 6~12개월 페이퍼 트레이딩 (유일한 순수 OOS)
- 새로운 자산 데이터: SPY/IWM에 동일 파라미터 교차 적용 (상대적으로 덜 오염)

**2층 — 오염을 인정하고 DSR을 보수적으로 재계산:**

- N=27이나 N=432가 아닌, 실질적 시행 횟수(N≈2,500+)로 DSR을 재계산
- 이전 단계의 모든 탐색(지표 선택, 전략 구조 설계 등)을 시행 횟수에 포함

**3층 — 경제적 논거로 방어:**

- 전략의 작동 원리가 특정 파라미터가 아닌 시장의 구조적 특성에 의존함을 확인
- 경제적 논거가 강할수록 파라미터 오염의 상대적 영향이 줄어듦

## 11. 검증 실행 계획: 3단계

### 11.1 개요

버퍼존 QQQ 전략을 기준으로 과최적화 여부를 최종 판단하기 위해 세 단계의 검증을 수행한다.

```
1단계: 파라미터 안정성 확인 → "특정 값에 의존하는가?"
  ↓
2단계: WFO 나쁜 윈도우 분석 → "시장 문제인가, 과최적화인가?"
  ↓
3단계: 교차 자산 검증       → "QQQ 암기인가, 범용적 전략인가?"
```

### 11.2 1단계: 432개 전체 파라미터 안정성 확인

**답하려는 질문:** "버퍼존 QQQ 전략이 특정 파라미터 조합에 의존하는가, 아니면 넓은 파라미터 영역에서 작동하는가?"

**통과 기준:**

- 432개 중 Calmar > 0인 조합이 과반수(216개+) 이상
- 최적 파라미터 인접 조합의 Calmar가 최적 대비 30% 이내 (0.21+ 이상)
- 히트맵에서 고원 형태가 관찰됨

### 11.3 2단계: WFO 나쁜 윈도우에서 전체 조합 분포 확인

**답하려는 질문:** "WFO에서 성과가 나빴던 구간이 시장 환경의 문제인가, 아니면 IS 최적화가 OOS에서 역효과를 낸 과최적화인가?"

**해석 기준:**

| IS 최적의 OOS 백분위 | 해석                                |
| -------------------- | ----------------------------------- |
| 40~60% (중앙 근처)   | 시장 환경의 문제. 과최적화 아님     |
| 20~40% (중앙 이하)   | 경미한 과최적화 의심                |
| 0~20% (하위권)       | 명확한 과최적화. IS 최적화가 역효과 |

### 11.4 3단계: 교차 자산 검증

**답하려는 질문:** "버퍼존 전략이 QQQ 과거 데이터를 암기한 것인가, 아니면 범용적 추세추종인가?"

파라미터를 완전 동결하여 QQQ 이외의 6개 자산에 적용했을 때 양(+)의 수익이 나오는지 확인한다.

### 11.5 3단계 이후의 의사결정 경로

```
1단계 통과 + 3단계 통과:
  → 버퍼존 QQQ 전략을 소규모 실전 투입 가능
  → 6~12개월 페이퍼 트레이딩 후 본격 운용

1단계 통과 + 3단계 실패:
  → QQQ 고유의 패턴에 의존하는 전략
  → QQQ 단일 자산으로만 제한적 운용 가능

1단계 실패:
  → 전략의 견고성 부족
  → 전략 자체를 폐기하고 새로운 가설에서 출발
```

### 12.3 주요 전환점 해설

**전환점 1 — "복잡성의 역설" 발견 (초기 진단)**

PBO/DSR 진단에서 QQQ(5P) → TQQQ(5P) → ATR WFO(7P)로 전략이 진화할수록 과최적화 지표가 악화되는 패턴이 발견되었다. 이 발견이 "가장 단순한 전략부터 검증한다"는 방향을 결정했다.

**전환점 2 — 1단계 "조건부 통과" 판정**

정량적 기준 5개 중 Pass 1 / Fail 2 / Warn 2로 형식적으로는 미달이었으나, Fail 2건이 모두 sell_buffer의 체계적 방향성에 기인하여 경제적으로 설명 가능했다. 이를 "조건부 통과"로 판정하고 3단계 교차 자산 검증으로 진행하는 근거가 되었다.

**전환점 3 — 3단계에서 hold_days 제거 결정과 그 부작용**

교차 검증의 순수성을 위해 hold_days를 제거(=0)하였다. 그러나 Calmar 33% 하락, MDD 12.7%p 악화가 확인되어 hold_days의 중요성을 재발견하게 되었다. 이 부작용이 §16 hold_days 심층 학술 연구의 직접적 동기이다.

**전환점 4 — DSR 처리 원칙 확립**

교차 자산 검증 계획 수립 중 "다른 ticker인데 왜 DSR을 합쳐서 통계?"라는 질문이 제기되어, "ticker별 별도 N, 별도 DSR" 원칙을 확립했다. 파라미터 고정 시 각 검증 자산의 N=1이므로 DSR 자체가 불필요하다는 결론에 도달했다 (§14.4 참조).

**전환점 5 — hold_days 방어 등급 상향**

학술 문헌 조사 결과, 4가지 독립적 메커니즘(단기 반전, 미시구조 노이즈, 노이즈 트레이더 과잉 반응, 가짜 돌파)이 확인 기간의 필요성을 지지. 방어 등급을 B~C에서 S(존재)/A(값)로 상향.

**전환점 6 — hold_days=2 실험 미달과 hold=3 재검토**

학술 근거에 기반하여 hold_days=2를 사전 결정하고 실험을 실행했으나, QQQ에서 Calmar 0.210으로 통과 기준(0.240)에 미달하였다. hold=2와 hold=3 사이의 MDD 차이가 12.1%p로 예상보다 크게 나타나, "2~5일은 고원"이라는 기존 가설이 QQQ에서는 성립하지 않을 가능성이 제기되었다. 이 미달이 고원 분석(전환점 7)의 직접적 동기이다.

**전환점 7 — 56회 고원 분석과 hold_days=3 최종 확정**

hold=0, 2, 3 세 개 포인트만으로는 "봉우리인지 고원인지" 판단할 수 없다는 문제를 인식하고, 7개 자산 × 8개 hold_days = 56회 백테스트를 실행했다. QQQ/SPY/GLD에서 고원 확인. 교집합 hold=3~5. hold=3은 교집합의 왼쪽 끝으로 확정.

**전환점 8 — 나머지 파라미터 고원 분석과 거래 수 함정 발견**

hold_days와 동일한 방법론을 sell_buffer, buy_buffer, ma_window에 적용하여 126회 백테스트를 실행했다. 4개 파라미터 전부 현재 값이 고원 안에 있음이 확인되었다. 특히 sell_buffer에서 "넓을수록 좋다"는 기존 관찰이 "거래 수 감소에 의한 환상"이었음이 밝혀졌다: sell=0.15에서 QQQ는 26년간 4거래밖에 없어 B&H에 수렴하는 상태였다. 이로써 B&H 벤치마크와의 비교가 직접적 동기가 되었고, QQQ/SPY/EFA에서 전략 우위가 확인되어 페이퍼 트레이딩 대상이 확정되었다.

**전환점 9 — recent_months 제거와 "복잡성의 역설" 재확인**

recent_months를 코드에서 삭제한 후, TQQQ Fixed의 CAGR이 6.95%→26.32%로 4배 개선. "동적 조정이 IS 과최적화를 유발했다"는 것이 밝혀지며, "단순한 고정 파라미터가 복잡한 동적 조정보다 우월하다"는 결론이 재확인.

**전환점 10 — 분할 매수매도 전략 기각과 고원의 올바른 해석**

고원 분석(§18)에서 ma_window 150~300이 고원임을 확인한 후, "고원 내 복수 포인트를 사용하면 더 강건해진다"는 가설로 3분할 전략(MA-150/200/250)을 설계·구현하였다. 그러나 백테스트에서 MDD가 4.51%p 악화되고 CAGR도 하락하여 모든 지표에서 단일 MA-200에 열등했다. WFO 데이터에서 QQQ MA가 11개 윈도우 전부 200으로 고정되어 있고, Dynamic(매 윈도우 최적 추적)이 Fixed(고정)보다 열등한 것이 확인되어, "고원 내 이동은 추적할 가치가 없는 노이즈"라는 결론에 도달했다. 고원의 올바른 해석은 "여러 포인트를 쓰자"가 아니라 "아무거나 하나 골라도 괜찮다는 자신감을 갖자"이다. 이것은 프로젝트의 일관된 교훈인 "복잡성의 역설"의 세 번째 확인이다 (첫째: ATR WFO 폐기, 둘째: recent_months 제거, 셋째: 분할 기각).

## 16. hold_days 파라미터 심층 분석

> §15.2에서 확인된 hold_days 제거의 부작용을 분석하면서 수행한 학술 조사의 결과이다.

### 16.1 파라미터 존재 자체의 정당성

hold_days(돌파 후 확인 기간)는 추세추종에서 "확인(confirmation)"이라 불리는 보편적 개념이다.

### 16.2 4가지 학술적 메커니즘

**메커니즘 1 — 단기 반전 효과 (Short-term Reversal):**

Lehmann(1990)은 주간 수익률 상위 종목이 다음 주에 0.35~0.55% 반전한다는 것을 문서화했다. Da, Liu & Schaumburg(2014)는 이 반전의 89%가 비펀더멘탈 요소에서 기인함을 분해하여 보여주었다.

**메커니즘 2 — 시장 미시구조 노이즈:**

Roll(1984)의 bid-ask bounce에 의한 음의 자기상관, Aït-Sahalia & Yu(2009)의 미시구조 노이즈 63% 설명. 1~3일 대기로 실제 수급 균형으로 안착.

**메커니즘 3 — 노이즈 트레이더 과잉 반응:**

De Long et al.(1990)의 긍정적 피드백 모형: 초기 움직임 → 모멘텀 트레이더 과잉 → 부분 수정 → 추세 재개. 확인 기간은 거품 국면을 건너뛰는 역할.

**메커니즘 4 — 가짜 돌파의 높은 비율:**

Bulkowski의 13,932개 차트 패턴 분석: 돌파 실패율 14~28%. 3일 연속 종가 유지 시 실패율 약 절반으로 감소.

### 16.3 Jegadeesh-Titman의 스킵 기간

J-T(1993)의 "2-12 모멘텀"(최근 1개월 제외)은 "단기 반전 노이즈를 건너뛰기 위한 스킵 기간"이다. hold_days는 일간 빈도의 스킵 기간으로, 동일한 학술적 논거에 기반한다.

### 16.4 최적 확인 기간에 대한 실증

- **Alvarez(2023):** SPY/QQQ에서 200일 MA 돌파 후 N일 연속 유지 조건을 N=1~10 테스트. N=2부터 N=10까지 모든 값이 N=1보다 우수.
- **Safari & Schmidhuber(2025):** 2~5일 확인 기간은 "평균 회귀에서 추세로 전환되는 경계"에 정확히 위치.

### 16.5 변동성에 따른 확인 기간의 실질적 의미

| 자산 | 일평균 변동성 | 3% 돌파 ÷ 변동성 | 해석               |
| ---- | ------------- | ---------------- | ------------------ |
| QQQ  | ~1.5%         | ~2.0일치         | 적절               |
| IWM  | ~1.8%         | ~1.7일치         | 노이즈 가능성 높음 |
| GLD  | ~0.8%         | ~3.8일치         | 의미 있는 움직임   |

### 16.6 방어 등급 재평가 — 최종

| 평가 항목            | 초기 등급 (§8.2) | 학술 조사 후 | 실험 확정 후 (최종) | 근거                                                                     |
| -------------------- | ---------------- | ------------ | ------------------- | ------------------------------------------------------------------------ |
| 파라미터 존재의 논거 | B~C              | **S**        | **S**               | 4가지 독립적 학술 메커니즘                                               |
| 값 = 3의 논거        | C                | B            | **A**               | 56회 고원 분석: 3개 자산 교집합(3~5) + 학술 범위(2~5) + 그리드 결과 수렴 |

## 20. recent_months 제거 영향 분석

> 분석일: 2026-03-14~15

> 데이터: git 이력 기반 OLD/NEW 비교

### 20.1 배경: recent_months가 무엇이었는가

recent_months는 최근 N개월의 청산 횟수에 따라 buy_buffer와 hold_days를 동적으로 증가시키는 메커니즘이었다. 전체기간 그리드 서치에서는 recent_months=0(동적 조정 비활성화)이 최적이어서, "결과 변동이 없을 것"이라 예상했다.

### 20.2 계획서의 전제가 틀렸다

**이 가정은 틀렸다.** 전체 기간 그리드 서치에서는 recent_months=0이 최적이었지만, **WFO 윈도우별 그리드 서치에서는 recent_months > 0이 최적으로 선택된 윈도우가 다수 존재했다.**

- QQQ Dynamic: 11개 윈도우 중 **4개**에서 recent_months > 0 (W0~W3)
- TQQQ Dynamic: 11개 윈도우 중 **8개**에서 recent_months > 0 (W0~W7)

### 20.4 변동 원인 분석

**원인 1 — 동적 조정이 IS 과최적화를 유발했다:** recent_months > 0이면 optimizer가 IS 데이터의 청산 패턴에 과적합된 공격적 파라미터 조합(예: sell_buffer=0.01)을 선택. OOS에서는 통하지 않았다.

**원인 2 — Fully Fixed 모드에서 치명적 영향:** OLD TQQQ Fixed는 ma=100, sell=0.01, recent_months=12로 전체 기간에 고정 적용. sell=0.01(1%)은 "추세 내 모든 되돌림에서 청산"을 의미하며, 동적 조정 없이는 재앙적. 이것이 이전 CAGR 6.95%, WFE -0.91의 직접적 원인이었다.

**원인 3 — 그리드 서치 차원 축소로 로버스트한 파라미터 선택:** 탐색 공간 4배 축소로 "동적 조정에 의존하는 파라미터" 대신 "단독으로 로버스트한 파라미터"가 선택.

> **후속 발견 (2026-04-01):** §20.3~20.5에서 사용한 WFE Robust 수치(2.09/2.38/2.39)가 EMA 연속성 버그에 의해 과대평가되었음이 밝혀졌다. recent_months 제거의 올바름과 Dynamic≈Fixed라는 핵심 발견은 불변이나, WFE 절대값에 기반한 낙관적 해석("IS의 2배가 OOS에서 재현")은 철회한다. 수정된 해석은 §20.8 참조.

| C     | OrderIntent: signal/rebalance 충돌 해소 | 포트폴리오 전체 | 중간                       |
| D     | Execution: SELL 선행 + 음수 현금 방지   | 포트폴리오 전체 | 소폭 (교정적)              |
| **E** | **WFO OOS EMA 연속성 보장**             | **WFO 전체**    | **대폭 (WFE Robust 급락)** |
| F     | Calmar MDD=0 계산 통일                  | WFO Calmar      | 미세                       |

#### 20.8.2 WFE Robust 급락의 메커니즘 (변경 E)

**핵심: 기존의 높은 WFE Robust는 버그에 의한 허상이었다. 수정 후 수치가 실제 전략 성능이다.**

`run_walkforward()`가 OOS 구간을 먼저 slice한 뒤 `pandas.ewm(span=200)`을 적용했다. EMA는 시리즈 첫 행부터 새로 시작하므로(`alpha ≈ 0.01`), OOS 첫날의 EMA가 전체 히스토리를 이어받지 못하고 리셋되었다. 수렴까지 약 200거래일이 필요하여, 24개월 OOS의 앞쪽 40%에서 EMA가 왜곡되었다.

인과 경로: EMA 리셋 → 가격 밀착 추종 → 밴드(upper/lower) 좁아짐 → 돌파 조건 충족 어려움 → **whipsaw 거래 자동 억제** → OOS Calmar 과대평가 → `wfe_calmar_robust = median(oos_calmar / is_calmar)` 부풀림.

IS 그리드 서치는 내부적으로 MA를 재계산하므로 IS Calmar는 불변. OOS Calmar만 정상화되어 WFE 비율이 급락했다.

수정 방법: 루프 진입 전 전체 `signal_df`에 모든 MA 윈도우를 사전 계산한 뒤 IS/OOS로 슬라이스. 논리적으로 완전한 수정이다.

#### 20.8.3 WFE Robust 변동 3단계 이력

| 지표                   | ①OLD (recent_months 존재) | ②중간 (recent_months 제거 직후) | ③최종 (EMA 버그 수정 후) |
| ---------------------- | ------------------------- | ------------------------------- | ------------------------ |
| QQQ Dynamic WFE        | 2.09                      | 2.09                            | **0.93**                 |
| QQQ Fixed WFE          | 2.69                      | 2.09                            | **0.93**                 |
| TQQQ Dynamic WFE       | 0.94                      | 2.38                            | **0.19**                 |
| TQQQ Fixed WFE         | -0.91                     | 2.39                            | **0.04**                 |
| QQQ gap_calmar_median  | —                         | 0.26                            | **-0.016**               |
| TQQQ gap_calmar_median | —                         | 0.62                            | **-0.088**               |

①→②(recent_months 제거): TQQQ WFE가 음수→양수로 개선된 것은 실제 버그(동적 조정 IS 과최적화) 수정에 의한 진짜 개선이었다.

②→③(EMA 버그 수정): 모든 WFE가 급락한 것은 OOS Calmar의 과대평가가 제거된 결과이다. ②의 높은 WFE(2.09/2.38/2.39)는 허상이었다.

#### 20.8.4 수정 후 WFO 해석

**QQQ — 양호, 단 강한 증거는 아님:**

WFE 0.93은 "IS 성과의 93%가 OOS에서 재현"으로, 실질적으로 양호한 수준이다. gap_calmar_median -0.016은 절대값이 0에 가까워 IS ≈ OOS 수준이며, 과최적화 우려가 없다. 다만 이전의 2.09처럼 "IS의 2배가 OOS에서 재현되는 강한 방어 논거"로 사용할 수는 없다. WFO는 전략 유효성의 **보조 확인 수단**으로 격하하고, 핵심 방어는 고원 분석(§17~18)과 교차 자산 검증(§15)이 담당한다.

**TQQQ — 실전 기대치 하향 필요:**

WFE 0.19(Dynamic)와 0.04(Fixed)는 "IS 대비 OOS 재현율이 매우 낮다"는 뜻이다. 이것은 레버리지 상품의 본질적 특성(whipsaw 손실이 3배 증폭)을 반영한 것일 수 있으나, TQQQ 단일 전체기간 백테스트 성과(CAGR 19.18%)가 실전에서 그대로 나올 것이라는 기대를 낮춰야 한다.

**Dynamic ≈ Fixed 관계 — 불변:**

EMA 버그 수정은 WFE의 "절대 수준"을 바꿨지만, Dynamic과 Fixed의 "상대 관계"는 바꾸지 않았다. 두 모드 모두 같은 버그의 영향을 같은 방향으로 받았기 때문이다. 따라서 "파라미터를 매번 바꾸든 고정하든 결과가 비슷하다 → 동적 변경은 불필요하다"는 결론(§21)은 유효하다.

**gap_calmar_median 부호 반전의 해석:**

기존 양수(0.26/0.62)는 EMA 리셋이 OOS Calmar를 부풀려 "OOS > IS"로 보이게 한 허상이었다. 수정 후 음수(-0.016/-0.088)는 "IS > OOS"를 의미하지만, 절대값이 0에 가까워 사실상 IS ≈ OOS 수준이다. "심각한 과최적화 발견"이 아니라 "기존 과대평가 허상이 제거된 것"이다.

#### 20.8.5 4P 동결 원칙에 대한 영향

이번 버그 수정은 4P 동결 원칙의 근거를 흔들지 않는다. 동결 원칙의 3가지 독립 근거 상태:

| 근거                  | 수정 전               | 수정 후                                 | 판정                 |
| --------------------- | --------------------- | --------------------------------------- | -------------------- |
| 고원 분석 (§17~18)    | 4P 전부 고원 안       | **불변** (단일 백테스트 엔진 변경 없음) | 유효                 |
| Dynamic ≈ Fixed (§21) | QQQ/TQQQ 모두 유사    | **불변** (상대 관계 변화 없음)          | 유효                 |
| WFO WFE Robust        | 2.09/2.38 (강한 증거) | 0.93/0.19 (**보조 증거로 격하**)        | 증거 약화, 결론 불변 |

WFE가 약해졌다는 것은 "전략의 실전 기대치를 더 보수적으로 잡아야 한다"는 뜻이지, "파라미터를 바꿔야 한다"는 뜻이 아니다. 오히려 TQQQ WFE 0.19는 "IS에서 최적을 찾아 적용해도 OOS에서 19%밖에 재현 안 된다"는 뜻이므로, **동적 변경의 무용성을 더 강하게 지지한다.**

## 21. WFO 동적 파라미터 변경을 하면 안 되는 이유

> **2026-04-01 수정:** EMA 연속성 버그 수정 후 WFE 절대값이 급락했으나(§20.8), 이 절의 핵심 논거인 "Dynamic ≈ Fixed"는 불변이다. WFE 하락은 오히려 동적 변경의 무용성을 강화한다 — IS에서 최적을 찾아도 OOS에서 재현율이 낮다면, 매번 새 최적을 찾아 적용할 실익이 없다.

### 21.1 이 프로젝트에서 직접 증명된 증거 3가지

**증거 1 — ATR WFO 전략의 실패:**

WFO 동적 파라미터 변경을 적용한 ATR WFO(TQQQ) 전략이 고정 파라미터 전략보다 과최적화 지표가 모두 나빴다.

| 전략         | 파라미터 변경 방식 | PBO  | DSR  | DSR Z-Score |
| ------------ | ------------------ | ---- | ---- | ----------- |
| 버퍼존 QQQ   | **고정**           | 0.40 | 0.78 | +0.771      |
| 버퍼존 TQQQ  | **고정**           | 0.45 | 0.65 | +0.395      |
| ATR WFO TQQQ | **WFO 동적**       | 0.65 | 0.35 | **-0.390**  |

ATR WFO의 DSR Z-Score가 음수(-0.390)라는 것은 파라미터를 동적으로 바꾼 전략이 운보다 못했다는 의미이다.

**증거 2 — QQQ WFO Dynamic ≈ Fixed 성과 동일:**

```
Dynamic (매 윈도우 재최적화): Stitched CAGR 11.33%
Fixed (첫 파라미터 고정):     Stitched CAGR 11.97%
```

매 윈도우마다 최적 파라미터를 새로 찾아서 적용해도, 처음 값을 고정해도 성과가 사실상 동일하며 Fixed가 오히려 약간 우위이다. 동적 변경이 가치를 더하지 않는다.

**증거 3 — TQQQ Fixed가 Dynamic보다 오히려 우월:**

```
TQQQ Dynamic: CAGR 24.84%, Calmar 0.347
TQQQ Fixed:   CAGR 26.27%, Calmar 0.423
```

Fixed가 Dynamic보다 CAGR, Calmar 모두 우수하다. 이전에는 Dynamic(21.4%)이 Fixed(7.0%)보다 3배 좋아 보였으나, 이것은 recent_months에 의한 IS 과최적화가 Fixed를 오염시킨 결과였다 (§20 참조). 오염 제거 후 **동적 변경이 가치를 더하지 않을 뿐 아니라 오히려 해를 끼칠 수 있다**는 것이 증명되었다.

또한 TQQQ PC 최대가 Dynamic 0.69, Fixed 0.68로 두 모드 모두 수익의 약 69%가 특정 구간(W9, 2023~2025 AI 랠리)에 집중되어 있다. 이것은 "적응이 잘 작동한 것"이 아니라 레버리지 상품의 구조적 특성이다.

### 21.2 학술적 근거

**탐색 횟수의 곱셈 효과:**

WFO를 실행하면 매 윈도우마다 그리드 서치를 수행한다. 실질적 시행 횟수는:

```
고정 파라미터:     N = 432 (1회 그리드 서치)
WFO 동적 (11윈도우): N ≤ 432 × 11 = 4,752 (보수적 상한*)
WFO 동적 (22윈도우): N ≤ 432 × 22 = 9,504 (보수적 상한*)
```

> \*Expanding Anchored WFO에서 윈도우들은 IS 데이터를 누적 공유하므로(W0의 IS ⊂ W1의 IS ⊂ ... ⊂ W10의 IS), 윈도우 간 그리드 서치 결과는 상당히 상관되어 있어 실질적 독립 시행 횟수는 이보다 낮다. 그러나 N이 커진다는 질적 결론은 유효하다.

Bailey et al.(2014)에 따르면 기대 최대 SR ≈ √(2 × ln(N))이므로, N이 커질수록 "운으로 좋은 결과가 나올 확률"이 높아진다.

**"적응"이 아니라 "노이즈 추종":**

"시장이 바뀌니까 파라미터도 바꿔야 한다"는 논리의 숨겨진 가정은 "IS에서 찾은 최적이 다음 OOS에서도 최적일 것"이다. 실제 시장의 regime change(2008 금융위기, 2020 코로나, 2022 금리 인상)는 급격하고 예측 불가능하다.

### 21.3 고원이 "바꾸지 않아도 된다"는 구조적 증거

182회 고원 분석(§17~18)에서 4개 핵심 파라미터 모두 고원 안에 위치함이 확인되었다. 고원 안에서 WFO 최적이 이리저리 움직이는 것은 **"시장이 바뀌었다"는 신호가 아니라 "고원 위의 노이즈"**이다. 성과가 비슷하므로 바꿀 이유가 없다.

## 22. WFO 점검 주기 및 방법

### 22.1 원칙

파라미터를 바꾸기 위해서가 아니라, **전략이 여전히 작동하는지 확인하기 위해** 점검한다.

### 22.2 점검 주기: 연 1회

| 항목          | 내용                                         |
| ------------- | -------------------------------------------- |
| 주기          | **연 1회** (매년 1월 또는 데이터 갱신 후)    |
| 방법          | run_walkforward.py 재실행 (최신 데이터 포함) |
| 파라미터 변경 | **하지 않는다** (점검 결과에 관계없이)       |

**왜 연 1회인가:** 이 전략의 평균 거래 빈도가 연 0.5~1회이므로, 6개월 이하 주기로 점검해도 새로운 정보가 없다. 더 자주 점검하면 "결과를 보고 바꾸고 싶은 유혹"이 커지고, 이것이 과최적화의 시작이다.

### 22.3 연 1회 점검 체크리스트

**체크 1 — Dynamic과 Fixed의 성과 괴리:**

괴리 없음(현재 상태): Dynamic CAGR ≈ Fixed CAGR → 정상. 고정 파라미터 유지.
괴리 발생: Dynamic CAGR >> Fixed CAGR → 경고. 그러나 파라미터를 바꾸지 않는다. §23 참조.

**체크 2 — 파라미터 추이의 고원 이탈:**

고원 안에서 변동(현재 상태): 정상. 고원 내 노이즈.
고원 밖으로 이탈: 심각한 경고. 전략 구조 자체의 유효성 재검토 필요.

**체크 3 — Stitched Equity의 장기 추세:**

우상향 유지: 정상.
횡보 또는 하락: 경고. §23의 정량적 기준과 대조.

### 22.4 점검 결과에 따른 행동

| 체크 결과     | 행동                               | 절대 하지 않는 것          |
| ------------- | ---------------------------------- | -------------------------- |
| 3개 모두 정상 | 파라미터 유지. 다음 해 재점검      | 파라미터 변경              |
| 1개 경고      | 기록만 남기고 관찰. 다음 해 재점검 | 파라미터 변경              |
| 2개 이상 경고 | §23(전략 폐기 기준) 검토           | 파라미터 미세 조정         |
| 고원 이탈     | §23.3(전략 재설계 기준) 검토       | WFO 최적으로 파라미터 교체 |

## 23. 전략 폐기 및 재설계 기준

### 23.1 원칙

파라미터 조정이 아니라 **전략 구조의 유효성을 판단**한다. 고원 분석(§17~18)에서 고원 내 어떤 값을 골라도 비슷하다는 것이 확인되었으므로, 성과 악화의 원인은 파라미터가 아니라 시장 구조 변화이다.

### 23.2 전략 폐기 기준

아래 조건 중 **2개 이상** 충족 시 전략 폐기를 검토한다.

| #   | 기준                                                | 임계값                                         | 근거                                             |
| --- | --------------------------------------------------- | ---------------------------------------------- | ------------------------------------------------ |
| 1   | **실전 Calmar가 B&H Calmar 이하로 3년 연속**        | 전략 Calmar < B&H Calmar × 3년                 | B&H보다 못하면 전략의 존재 이유가 없다           |
| 2   | **실전 MDD가 백테스트 MDD의 1.5배 초과**            | 예: QQQ 실전 MDD > -36.49% × 1.5 = -54.7%      | 모델 가정 자체가 틀렸음을 시사                   |
| 3   | **WFO Dynamic과 Fixed의 성과 괴리가 3년 연속 확대** | Dynamic Calmar > Fixed Calmar × 2.0 (3년 연속) | 고정 파라미터로는 대응 불가능한 수준의 시장 변화 |
| 4   | **WFO 최적 파라미터가 고원 밖으로 3회 연속 이탈**   | 4개 핵심 파라미터 중 2개 이상이 고원 밖        | 기존 고원이 더 이상 유효하지 않음                |

**"3년"과 "3회"의 이유:** 추세추종 전략은 구조적으로 횡보장에서 손실을 보므로, 1~2년의 부진은 정상 범위이다.

### 23.3 전략 재설계 기준

```
단계 1: 전략을 중단한다 (포지션 청산)
단계 2: 원인을 분석한다
  → "모든 추세추종이 안 되는 시장인가?"
  → "200일 MA 기반 추세추종만 안 되는 것인가?"
  → "우리 전략의 특수한 문제인가?"
단계 3: 원인에 따라 결정한다
  → 모든 추세추종이 안 됨: 전략 폐기, 다른 전략 유형 탐색
  → MA 기반만 안 됨: 전략 구조 재설계 (새로운 가설에서 출발)
  → 특수한 문제: 해당 문제만 수정 (예: 슬리피지 모델 갱신)
```

**재설계 시 주의:** 새 전략을 설계할 때 기존 데이터를 사용하면 "이미 답을 본" 오염이 재발한다. 가능하면 새로운 자산이나 새로운 기간의 데이터에서 출발해야 한다.

### 23.4 전략 폐기가 아닌 것

| 상황                              | 행동                   | 이유                              |
| --------------------------------- | ---------------------- | --------------------------------- |

## 25. 분할 매수매도 전략 검토 및 기각

> 검토일: 2026-03-15
> 배경: §18 고원 분석에서 ma_window 150~300이 고원임을 확인한 후, 고원 내 복수 포인트(MA-150/200/250)를 사용하는 3분할 전략을 설계·구현·백테스트하였다.
> 결론: **기각. 단일 MA-200이 분할보다 모든 지표에서 우수.**

### 25.1 분할 전략의 설계

182회 고원 분석(§17~18)에서 ma_window 150~300이 고원임이 확인되자, "고원 내 복수 포인트를 사용하면 단일 포인트 의존을 줄일 수 있다"는 가설로 3분할 전략을 설계하였다.

| 파라미터      | 트랜치 1 | 트랜치 2 (기준) | 트랜치 3 |
| ------------- | -------- | --------------- | -------- |
| **ma_window** | **250**  | **200**         | **150**  |
| buy_buffer    | 0.03     | 0.03            | 0.03     |
| sell_buffer   | 0.05     | 0.05            | 0.05     |
| hold_days     | 3        | 3               | 3        |
| 가중치        | 33%      | 34%             | 33%      |

ma_window만 변경하고 나머지 3개 파라미터는 전 트랜치 동일. 공유 자본 모델(shared_cash)로 구현하여 트랜치 간 자본을 동적 배분.

### 25.2 백테스트 결과: 분할이 모든 지표에서 열등

| 지표    | 단일 MA-200 (QQQ) | 분할 150/200/250 (QQQ) | 차이               |
| ------- | ----------------- | ---------------------- | ------------------ |
| CAGR    | 10.93%            | 10.69%                 | **-0.24%p**        |
| MDD     | -36.49%           | -41.00%                | **-4.51%p (악화)** |
| Calmar  | 0.30              | 0.26                   | **-0.04**          |
| 거래 수 | 14                | 43                     | +29 (비용 3배)     |

| 지표   | 단일 MA-200 (TQQQ) | 분할 150/200/250 (TQQQ) | 차이    |
| ------ | ------------------ | ----------------------- | ------- |
| CAGR   | 19.18%             | 18.73%                  | -0.45%p |
| MDD    | -89.30%            | -89.60%                 | -0.30%p |
| Calmar | 0.21               | 0.21                    | 0.00    |

QQQ에서 분할의 MDD가 단일보다 4.51%p 더 깊다. TQQQ에서도 개선 효과 없음.

### 25.3 기각 근거 1 — 시그널 상관관계가 너무 높다

MA-150과 MA-250은 기초 가격 데이터의 약 60%를 공유한다. Robert Carver(前 Man AHL)의 EWMAC 시스템에서 인접 속도 추세 시그널 간 상관관계는 약 0.88~0.93이며, 0.95 이상이면 변형을 추가하지 않는다고 명시한다.

3개 전략의 쌍별 상관관계가 0.95일 때 포트폴리오 표준편차 감소는 **단 1.7%**이다. 반면 3개 비상관(ρ=0) 전략이면 42% 감소. 실질적 독립 베팅 수는 약 1.03으로, 사실상 1개 시그널과 구분이 안 된다.

실제 데이터가 이를 증명한다: 현재 3개 트랜치 모두 **동일한 날짜(2025-05-16)에 동일한 가격(QQQ: $520.36)으로 진입**하여 보유 중이다. "시간적 분산"이 이론적으로 존재하지만, 장기 추세에서는 대부분 동시에 반응한다.

### 25.4 기각 근거 2 — MDD 악화의 구조적 원인

분할의 MDD가 단일보다 악화된 것은 구조적으로 예측 가능한 결과이다.

```
급격한 하락 시:
  MA-150 (빠른 트랜치): 먼저 청산 → 1/3 자본 보호
  MA-200 (중간 트랜치): 며칠 후 청산 → 1/3 자본 추가 하락 노출
  MA-250 (느린 트랜치): 가장 늦게 청산 → 1/3 자본 최대 하락 경험

  단일 MA-200: 타협적 속도로 전체 자본을 한 번에 청산
```

느린 트랜치(MA-250)가 1/3 자본을 추가 하락에 노출시키면서, 빠른 트랜치(MA-150)의 조기 청산 이득을 상쇄하고도 남는다. 이것은 동일 자산에서 멀티 트랜치 이진(binary) 시스템의 알려진 약점인 **부분 노출 문제(partial exposure problem)**이다.

Carver의 프레임워크에서는 단일 자산의 복수 예측을 **하나의 가중 평균 예측으로 결합한 뒤 하나의 포지션 사이즈를 결정**한다. 자본을 별도 버킷으로 분리하지 않는다.

### 25.5 기각 근거 3 — WFO가 "고원 내 이동은 노이즈"임을 증명

"고원 내에서 최적이 이동할 수 있으니 분할이 보험이 된다"는 가설을 WFO 데이터로 검증하였다.

**QQQ WFO Dynamic 파라미터 추이:**

```
W0:  MA=200, buy=0.03, sell=0.05, hold=3
W1:  MA=200, buy=0.01, sell=0.05, hold=5  ← 유일한 변동
W2~W10: MA=200, buy=0.03, sell=0.05, hold=3  (9개 연속 동일)
```

**QQQ에서 MA는 11개 윈도우 전부 200이다.** 단 한 번도 150이나 250이 최적으로 선택된 적이 없다. MA-150과 MA-250에 자본의 2/3를 배분하는 것은 한 번도 "최적"이었던 적 없는 파라미터에 투자하는 것이다.

**TQQQ WFO Dynamic 파라미터 추이:**

```
W0:  MA=150, buy=0.03, sell=0.05, hold=5
W1:  MA=100, buy=0.03, sell=0.01, hold=0  ← 분할 범위 밖
W2~W10: MA=150, buy=0.03, sell=0.05, hold=3~5  (150으로 수렴)
```

TQQQ에서 MA가 이동한 유일한 사례(W1)는 MA=100으로, 150/200/250 분할이 커버하지 못하는 범위이다.

**결정적 증거 — Dynamic vs Fixed 성과 비교:**

| 모드                              | QQQ Calmar | TQQQ Calmar |
| --------------------------------- | ---------- | ----------- |
| Dynamic (매 윈도우 최적으로 변경) | 0.318      | 0.347       |
| Fixed (처음 값 고정)              | **0.436**  | **0.423**   |

"고원 내에서 최적이 이동하니까 대비해야 한다"는 논리를 극단까지 밀어붙이면, 매 윈도우마다 최적 파라미터로 바꿔주는 Dynamic이 최선이어야 한다. 그러나 **Fixed가 Dynamic보다 Calmar가 37%(QQQ), 22%(TQQQ) 높다.** 고원 내 이동은 추적할 가치가 없는 노이즈이며, 노이즈에 대한 "보험"인 분할도 가치가 없다.

### 25.6 기각 근거 4 — 숨겨진 과최적화 위험

분할 설계는 Harvey, Liu & Zhu(2016)가 말하는 연구자 자유도를 추가한다.

| 암묵적 선택  | 선택한 값   | 다른 가능한 값           |
| ------------ | ----------- | ------------------------ |
| 트랜치 수    | 3           | 2, 4, 5                  |
| 구체적 MA 값 | 150/200/250 | 160/200/240, 100/200/300 |
| 가중치       | 33:34:33    | 25:50:25, 40:20:40       |
| 간격         | 50일        | 25일, 75일, 100일        |

각 선택이 명시적으로 최적화되지 않았더라도, 연구자가 테스트할 수 있었던 구성의 공간을 확장한다.

### 25.7 기각 근거 5 — 고원의 올바른 해석

고원이 알려주는 것과 올바른 행동:

| 고원의 의미              | 잘못된 해석              | 올바른 해석                               |
| ------------------------ | ------------------------ | ----------------------------------------- |
| "MA=150~300 어디든 비슷" | "여러 개를 쓰면 더 안전" | **"아무거나 1개 골라도 안전"**            |
| 성과가 파라미터에 둔감   | 분산 기여가 크다         | **시그널이 너무 비슷해서 분산이 안 된다** |

고원이 넓고 편평할수록 그 안의 시그널들은 더 비슷하게 행동하고, 분할의 분산 효과는 더 줄어든다. **아이러니하게도, 고원이 잘 확인될수록 분할의 근거가 약해진다.**

### 25.8 분산이 필요하다면 — 올바른 방향

학술 문헌이 제시하는 분산 효과의 계층 구조:

| 방법                                         | 시그널 간 상관관계 | 독립 베팅 수 | 효과            |
| -------------------------------------------- | ------------------ | ------------ | --------------- |
| **교차 자산 추세추종** (QQQ+SPY+GLD)         | ~0.09              | ~2.75        | **가장 효과적** |
| 전략 유형 다변화 (추세+캐리+밸류)            | ~0.15              | ~2.0+        | 효과적          |
| 넓은 범위 파라미터 분산 (MA-50/200/400)      | ~0.17              | ~1.8         | 중간            |
| **좁은 범위 파라미터 분산 (MA-150/200/250)** | **~0.93**          | **~1.03**    | **거의 무효**   |

§15에서 이미 교차 자산 검증을 완료했고, §19에서 SPY(Calmar 0.43)가 QQQ(0.30)보다 전략 적합성이 높음을 확인했다. 파라미터를 쪼개는 대신 **자산을 추가(QQQ + SPY 병행)**하는 것이 분산 효과가 약 25배 크다.
