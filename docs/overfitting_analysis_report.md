## 1. 분석 배경

### 1.1 분석 대상

QQQ 데이터를 시그널 소스로 사용하는 이동평균 밴드 기반 추세추종 전략 3종에 대해 과최적화 여부를 진단한다. 전략은 다음 순서로 진화하였다.

```
버퍼존 (QQQ) → 버퍼존 (TQQQ) → 버퍼존 ATR (TQQQ) + WFO
```

### 1.2 분석에 사용된 학술 프레임워크

본 분석은 다음 학술 연구에 기반한다.

- **Bailey, Borwein, López de Prado & Zhu (2014)** — "Pseudo-Mathematics and Financial Charlatanism": 백테스트 과최적화의 수학적 필연성 증명
- **Bailey & López de Prado (2014)** — "The Deflated Sharpe Ratio": 다중 검정과 비정규 분포를 보정하는 샤프 비율 검정
- **Bailey, Borwein, López de Prado & Zhu (2016)** — "The Probability of Backtest Overfitting": PBO(백테스트 과최적화 확률) 지표와 CSCV(조합적 대칭 교차검증) 도입
- **Harvey, Liu & Zhu (2016)** — "…and the Cross-Section of Expected Returns": 316개 팩터에 대한 다중 검정 분석
- **Harvey & Liu (2015)** — "Backtesting": 헤어컷 샤프 비율과 비선형 다중 검정 보정

### 1.3 과최적화의 핵심 원리

기대 샤프 비율이 0인(예측력 없는) 전략이라도, N개의 독립 시행 중 최선을 선택하면 운으로 달성 가능한 기대 최대 샤프 비율은 다음과 같다.

```
E[max SR] ≈ √(2 × ln(N))
```

| 시행 횟수(N) | 운으로 가능한 최대 SR | 해석 |
|---|---|---|
| 27 | ~2.6 | 파라미터 3개 단순 조합 |
| 432 | ~3.5 | 버퍼존 QQQ/TQQQ 그리드 서치 |
| 1,728 | ~3.9 | ATR WFO 그리드 서치 |

관측된 샤프 비율이 이 "운의 벽"보다 낮다면, 해당 전략의 성과가 순전히 운에 의한 것일 가능성을 배제할 수 없다.

## 2. 핵심 결론 요약

### 2.1 전략별 판정

| 전략 | 과최적화 위험도 | PBO | DSR | 판정 |
|---|---|---|---|---|
| 버퍼존 (QQQ) | **중간** | 0.40 | 0.78 | 추가 검증 후 판단 가능 |
| 버퍼존 (TQQQ) | 중간~높음 | 0.45 | 0.65 | QQQ 검증 후 자동 결정 |
| 버퍼존 ATR (TQQQ) + WFO | **높음** | 0.65 | 0.35 | 현 상태로 실전 투입 부적절 |

### 2.2 전략 선택 근거

버퍼존 QQQ를 검증의 기준점으로 선정한 이유는 다음과 같다.

- 세 전략 중 PBO(0.40)와 DSR(0.78)이 가장 양호
- 파라미터 수(5개)와 조합 수(432개)가 가장 적어 과최적화 위험이 최저
- 그리드 상위 4개 결과의 Calmar 범위(0.292~0.301)가 안정적
- TQQQ 전략은 시그널이 동일하고 매매 대상만 다르므로, QQQ 검증이 선행되어야 의미가 있음
- ATR WFO는 DSR Z-Score가 음수(-0.390)이므로 추가 파라미터 도입이 오히려 성과를 악화시킨 증거

### 2.3 전략 진화가 보여준 역설

```
전략 진화:    QQQ → TQQQ → ATR+WFO

파라미터 수:  5개 → 5개 → 7개        (증가)
조합 수:      432 → 432 → 1,728      (4배 증가)
PBO:          0.40 → 0.45 → 0.65     (악화)
DSR:          0.78 → 0.65 → 0.35     (급락)
```

"더 정교한 전략"이 "더 많은 파라미터"를 의미할 때, 과최적화 위험이 기하급수적으로 증가한다. 복잡성을 추가하는 것은 성과를 개선한 것이 아니라 운으로 좋은 결과가 나올 확률만 높인 것이다.

## 3. 전략별 과최적화 진단

### 3.1 버퍼존 (QQQ) — 위험도: 중간

**긍정적 요소:**

- 그리드 상위 4개의 Calmar(0.292~0.301)가 매우 안정적이며, "봉우리"가 아닌 "고원"의 신호
- 1위 파라미터에서 recent_months=0으로, 동적 조정 메커니즘을 사용하지 않아 실질 자유도가 낮음
- 경제적 논거(추세추종, 200일 이동평균)가 학술적으로 지지됨
- 슬리피지(±0.3%) 반영, 미래 참조 방지(다음 영업일 시가 체결)가 적용됨

**우려 요소:**

- 432개 조합 전체에 대한 파라미터 안정성(히트맵)이 미확인
- 총 거래 14회로 통계적 검정력이 부족 (71% 승률의 유의성 미달)
- 전체 기간(1999~2026)에 대한 그리드 서치로 진정한 OOS 기간이 존재하지 않음
- DSR 0.78은 95% 신뢰 수준(0.95) 미달

### 3.2 버퍼존 (TQQQ) — 위험도: 중간~높음

**진단:**

QQQ와 로직이 동일하고 매매 대상만 다르므로, 과최적화 진단의 핵심은 QQQ 전략에 있다. 다만 TQQQ 고유의 추가 위험이 존재한다.

- 최적 MA가 QQQ(200)와 TQQQ(150)에서 다르게 나타남 → 매매 대상에 의해 최적 파라미터가 변하므로, 시그널의 범용성에 의문
- MDD -86.96%는 실전에서 회복이 극히 어려운 수준
- PBO 0.45는 동전 던지기(0.50)에 근접하여, 파라미터 최적화의 가치가 거의 없음을 시사

### 3.3 버퍼존 ATR (TQQQ) + WFO — 위험도: 높음

**핵심 문제점:**

- PBO 0.65: IS 최적이 OOS에서 중앙값 이하일 확률 65%. 최적화를 할수록 오히려 나빠지는 상태
- DSR 0.35, Z-Score -0.390: 관측 SR(0.707) < 벤치마크 SR(0.800). 1,728개 조합 탐색의 기대 최대 SR보다 관측값이 낮다는 것은, 전략이 운보다도 못하다는 의미
- OOS CAGR 표준편차(30.07%)가 평균(16.69%)의 약 2배로 성과가 극단적으로 불안정
- WFO 11개 윈도우 중 OOS Calmar 기준 양수:음수 = 5:6
- Expanding Window 사용으로 초기 데이터(닷컴 버블)가 모든 윈도우에 누적 영향

**WFO 윈도우별 파라미터 불안정성:**

```
Window 0:   MA=150, 매수=0.05, 매도=0.05, 유지=5, 조정=8
Window 1~5: MA=200, 매수=0.01, 매도=0.03, 유지=0, 조정=0  (5개 연속 동일)
Window 6:   MA=200, 매수=0.03, 매도=0.03, 유지=2, 조정=12  (갑자기 변경)
Window 7~8: MA=200, 매수=0.03, 매도=0.05, 유지=2, 조정=12
Window 9~10: MA=200, 매수=0.03, 매도=0.03, 유지=3, 조정=0  (또 변경)
```

파라미터가 윈도우마다 변하는 것은 전략의 시간 안정성이 낮다는 증거이며, 누적된 새 데이터가 기존 최적을 뒤집을 만큼 시장 구조가 달라졌음을 시사한다.

## 4. CSCV/PBO 분석 해석

### 4.1 PBO의 의미

PBO(Probability of Backtest Overfitting)는 "IS에서 최적으로 선택한 전략이 OOS에서 중앙값 이하 성과를 보일 확률"이다. CSCV는 데이터를 6개 블록으로 나누어 C(6,3)=20가지 방법으로 IS/OOS를 분할하고, 각 분할에서 IS 최적의 OOS 상대 순위를 측정한다.

### 4.2 전략별 PBO 해석

| 전략 | PBO | 해석 |
|---|---|---|
| 버퍼존 (QQQ) | 0.40 | 20회 중 8회 실패. 학술적 경고 수준(0.50) 미만이나 안심 불가 |
| 버퍼존 (TQQQ) | 0.45 | 20회 중 9회 실패. 동전 던지기에 근접. 최적화의 가치 의심 |
| ATR WFO (TQQQ) | 0.65 | 20회 중 13회 실패. 명확한 과최적화. 최적화가 역효과 |

### 4.3 PBO 해석 시 주의사항

PBO는 "모든 구간에서 성과가 좋아야 한다"는 기준이 아니다. PBO가 측정하는 것은 "IS 최적이 OOS에서 상대적으로 어디에 위치하는가"이다.

시장이 나빠서 모든 조합이 다 같이 손실을 보는 구간에서, IS 최적도 손실을 본다면, 이것은 PBO에 큰 영향을 주지 않는다 (IS 최적의 상대 순위가 중앙 근처이므로). PBO가 높아지는 것은 "IS 최적이 OOS에서 유독 나쁜" 경우에만 해당한다.

## 5. DSR 분석 해석

### 5.1 DSR의 의미

DSR(Deflated Sharpe Ratio)은 두 가지 왜곡을 보정한 샤프 비율 검정이다.

1. **다중 검정에 의한 선택 편향**: N개 조합 중 최선을 선택하면 SR이 부풀려짐
2. **비정규 분포**: 왜도와 첨도를 반영하여 SR의 통계적 유의성을 재계산

DSR > 0.95이면 95% 신뢰 수준에서 전략의 SR이 다중 검정 보정 후에도 유의미하다고 판단한다.

### 5.2 전략별 DSR 해석

| 전략 | DSR | SR 관측 | SR 벤치마크 | Z-Score | 해석 |
|---|---|---|---|---|---|
| 버퍼존 (QQQ) | 0.78 | 0.874 | 0.694 | +0.771 | 95% 미달이나 방향은 양(+) |
| 버퍼존 (TQQQ) | 0.65 | 0.786 | 0.693 | +0.395 | 95% 미달, 약한 양(+) |
| ATR WFO (TQQQ) | 0.35 | 0.707 | 0.800 | **-0.390** | **관측값이 운의 벽 미달** |

ATR WFO의 Z-Score가 음수라는 것은, 1,728개 조합을 탐색한 결과의 최고 SR이 순전히 운으로 달성 가능한 기대 최대 SR보다 낮다는 의미다. 이것은 가장 강한 과최적화 경고 신호이다.

### 5.3 DSR 계산 시 "진짜 N" 문제

DSR의 핵심 입력값은 "시도한 전략의 수(N)"이다. 그러나 실질적 N은 형식적 그리드 서치 조합 수보다 크다.

```
실질적 시행 횟수 추정:
  EMA vs SMA 선택                    → +2
  지표 유형 탐색 (밴드 구조 등)        → +α
  버퍼존 QQQ 그리드 서치               → +432
  TQQQ로 매매 대상 변경               → +432
  ATR 추가                           → +1,728
  WFO 설정 탐색                       → +추가 시행
  파라미터 단순화 탐색                 → +27
  ────────────────────────────────────
  보수적 추정: N ≈ 2,500~3,000+
```

DSR을 정직하게 계산하려면, 전략 개발 과정에서의 모든 암묵적 시행을 포함해야 한다. N=432가 아닌 N=2,500+으로 재계산하면 벤치마크 SR이 상승하여 DSR은 더 낮아진다.

## 6. WFO 성과 분석과 해석 원칙

### 6.1 WFO의 궁극적 목표

WFO(Walk-Forward Optimization)는 "모든 구간에서 돈을 벌어야 한다"는 도구가 아니다. WFO의 목표는 **"이 전략이 미래에 돈을 벌 가능성이 있는가"를 판단하는 진단 도구**이다.

추세추종 전략은 구조적으로 횡보장에서 손실을 보므로, 일부 OOS 윈도우의 손실 자체는 과최적화의 증거가 아니다.

### 6.2 WFO 윈도우 성과의 올바른 해석법

성과가 나빴던 OOS 윈도우를 분석할 때 구분해야 하는 두 가지 시나리오가 있다.

**시나리오 A — 시장이 나빠서 전체 조합이 다 안 좋은 경우:**

해당 OOS 기간에 전체 432개(또는 1,728개) 조합의 수익률 분포를 확인했을 때, 대부분이 음(-)이고 IS 최적 조합이 그 분포의 중앙 근처에 위치한다면, 이것은 **시장 환경의 문제**이지 과최적화가 아니다.

**시나리오 B — IS 최적이 유독 안 좋은 경우:**

전체 조합 분포에서 양(+)인 조합이 존재하는데 IS 최적만 하위권에 위치한다면, 이것은 **과최적화의 증거**이다. IS에서의 최적화가 과거 노이즈에 맞춰진 것이다.

이 구분을 위해 각 OOS 윈도우에서 전체 조합의 수익률 분포를 뽑아, IS 최적의 백분위 순위를 확인하는 추가 분석이 필요하다.

### 6.3 ATR WFO 윈도우별 성과 진단

| Window | OOS CAGR | OOS Calmar | 판정 |
|---|---|---|---|
| 0 | -5.58% | -0.273 | 손실 |
| 1 | -9.88% | -0.168 | 손실 |
| 2 | +65.54% | +1.740 | 대박 |
| 3 | -20.88% | -0.318 | 큰 손실 |

## 8. 경제적 논거 평가

### 8.1 전략의 경제적 기반

버퍼존 전략의 핵심 논리: "QQQ가 장기 이동평균(200일 EMA) 위로 확실히 올라서면 추세가 시작된 것으로 판단하여 진입하고, 아래로 확실히 내려가면 추세가 끝난 것으로 판단하여 청산한다."

이것은 **모멘텀/추세추종**의 고전적 구현이며, 다음 학술 연구에 의해 지지된다.

- **Jegadeesh & Titman (1993, Journal of Finance)**: 3~12개월 모멘텀의 초과 수익을 최초로 문서화
- **Asness, Moskowitz & Pedersen (2013)**: 주식, 채권, 통화, 원자재 8개 시장에서 모멘텀의 범용성 확인
- **Moskowitz, Ooi & Pedersen (2012, Journal of Financial Economics)**: 58개 선물 시장에서 시계열 모멘텀(과거 수익률 부호 기반)의 유의미한 수익 보고
- **행동경제학적 기반**: 투자자의 과소반응(underreaction)과 과잉반응(overreaction)에 의한 추세 형성. 인간 심리에 기반하므로 구조적으로 지속될 가능성이 높음

### 8.2 경제적 논거의 방어력과 한계

경제적 논거는 전략의 **방향(direction)**을 지지하지만, 특정 **좌표(coordinates)**를 지지하지는 않는다.

| 주장 | 방어 등급 | 근거 |
|---|---|---|
| "추세추종이 작동한다" | **S등급** | 수십 년간 다수 시장에서 재현된 학술적 증거 |
| "200일 이동평균을 사용한다" | **A등급** | 기관/개인 투자자 사이에서 가장 널리 사용되는 추세 필터. 자기실현적 예언 효과 존재 |
| "매수버퍼=3%, 매도버퍼=5%가 최적이다" | **B등급** | 특정 값이어야 하는 경제적 이유를 설명하기 어려움 |
| "hold_days=3이 최적이다" | **B~C등급** | 데이터에서 발견된 값. 경제적 방어 거의 불가 |

### 8.3 "시장 참여자들도 동일하게 생각한다"의 양면성

200일 이동평균은 실제로 가장 널리 사용되는 기술적 지표 중 하나이다. 이것은 두 가지 상반된 함의가 있다.

**긍정적 측면 — 자기실현적 예언(self-fulfilling prophecy):** 많은 트레이더가 200일 MA를 보고 매매 결정을 내리므로, 가격이 실제로 200일 MA 근처에서 반응하는 경향이 있다. 이는 전략의 경제적 기반을 강화한다.

**부정적 측면 — 혼잡 거래(crowded trade):** 동일한 신호를 보는 참여자가 많으면, 진입/청산 시점이 집중되어 슬리피지가 증가하고, 급격한 반전 시 동시 청산으로 손실이 증폭될 수 있다.

### 8.4 결론

경제적 논거만으로 과최적화를 면제받을 수는 없지만, **과최적화의 확률을 상당히 낮춰준다.** 특히 전략이 특정 파라미터 값이 아닌 시장의 구조적 특성(모멘텀 프리미엄)에 의존하는 경우, 파라미터 오염의 영향이 상대적으로 줄어든다.

## 9. 긍정적 요소

과최적화 우려에도 불구하고, 다음 요소들은 전략의 잠재적 유효성을 지지한다.

1. **경제적 논거가 존재**: 무작위 데이터 마이닝이 아닌 합리적 가설(추세추종)에서 출발한 전략
2. **상위 파라미터의 안정성**: QQQ 그리드 상위 4개의 Calmar 범위가 0.292~0.301로 매우 좁음
3. **슬리피지 반영**: ±0.3%의 거래 비용을 적용하여 비현실적 가정을 배제
4. **미래 참조 방지**: 다음 영업일 시가 체결로 look-ahead bias를 구조적으로 차단
5. **벤치마크 대비 우위**: 버퍼존 QQQ의 Calmar(0.30)는 B&H QQQ(0.12)의 2.5배
6. **1위 파라미터가 가장 단순**: recent_months=0으로 동적 조정 미사용은 복잡성 최소화의 신호

## 10. "이미 답을 본" 오염 문제

### 10.1 문제의 본질

432개 그리드 서치를 실행한 순간, 그 데이터셋에 대한 "순수한 OOS 검증"은 영원히 불가능해졌다. 파라미터를 3개로 줄여 27개 조합만 탐색하더라도, 연구자의 뇌에는 432개 탐색의 결과가 내재되어 있다. 이것은 학술적으로 **"연구자 자유도에 의한 데이터 오염(researcher degrees of freedom)"** 또는 **"암묵적 다중 검정(implicit multiple testing)"**이라고 한다.

### 10.2 해결 접근법

데이터 오염을 완전히 제거할 수는 없지만, 영향을 측정하고 최소화하는 방법이 있다.

**1층 — 연구자가 본 적 없는 데이터로 검증:**

- 새로운 시간 데이터: 파라미터를 동결하고 6~12개월 페이퍼 트레이딩 (유일한 순수 OOS)
- 새로운 자산 데이터: SPY/IWM에 동일 파라미터 교차 적용 (상대적으로 덜 오염)

**2층 — 오염을 인정하고 DSR을 보수적으로 재계산:**

- N=27이나 N=432가 아닌, 실질적 시행 횟수(N≈2,500+)로 DSR을 재계산
- 이전 단계의 모든 탐색(지표 선택, 전략 구조 설계 등)을 시행 횟수에 포함

**3층 — 경제적 논거로 방어:**

- 전략의 작동 원리가 특정 파라미터가 아닌 시장의 구조적 특성에 의존함을 확인
- 경제적 논거가 강할수록 파라미터 오염의 상대적 영향이 줄어듦

## 11. 검증 실행 계획: 3단계

### 11.1 개요

버퍼존 QQQ 전략을 기준으로 과최적화 여부를 최종 판단하기 위해 세 단계의 검증을 수행한다. 각 단계는 독립적인 질문에 답하며, 순서대로 수행하는 것을 권장한다.

```
1단계: 파라미터 안정성 확인 → "특정 값에 의존하는가?"
  ↓
2단계: WFO 나쁜 윈도우 분석 → "시장 문제인가, 과최적화인가?"
  ↓
3단계: 교차 자산 검증       → "QQQ 암기인가, 범용적 전략인가?"
```

### 11.2 1단계: 432개 전체 파라미터 안정성 확인

**답하려는 질문:** "버퍼존 QQQ 전략이 특정 파라미터 조합에 의존하는가, 아니면 넓은 파라미터 영역에서 작동하는가?"

**필요 데이터:** 이미 보유한 grid_results.csv (432개 조합의 전체 성과)

**수행 방법:**

1. **전체 분포 히스토그램**: 432개 조합의 Calmar 분포를 히스토그램으로 시각화한다. 대부분이 양(+)이면 전략 자체가 견고한 것이고, 1위만 돌출되어 있으면 과최적화의 증거이다.

2. **파라미터별 히트맵**: MA=200을 고정한 상태에서 buy_buffer × sell_buffer의 2차원 히트맵을 그린다. 최적 파라미터 주변이 "고원(넓고 완만한 영역)"인지 "봉우리(좁고 급격한 피크)"인지 시각적으로 확인한다.

3. **근접 파라미터 성과 확인**: 최적(MA=200, buy=0.03, sell=0.05, hold=3)에서 한 단계만 변경한 조합의 Calmar를 확인한다.

   ```
   최적: buy=0.03, sell=0.05 → Calmar 0.301
   인접: buy=0.01, sell=0.05 → Calmar ?
   인접: buy=0.05, sell=0.05 → Calmar ?
   인접: buy=0.03, sell=0.03 → Calmar ?
   인접: buy=0.03, sell=0.01 → Calmar ?
   ```

4. **hold_days / recent_months 제거 실험**: hold_days=0, recent_months=0으로 고정하여 파라미터를 5개→3개로 줄인 버전을 실행한다. 조합이 432→27개로 감소하여 DSR이 개선되는지 확인한다.

**통과 기준:**

- 432개 중 Calmar > 0인 조합이 과반수(216개+) 이상
- 최적 파라미터 인접 조합의 Calmar가 최적 대비 30% 이내 (0.21+ 이상)
- 히트맵에서 고원 형태가 관찰됨

**소요 시간 추정:** 이미 보유한 데이터 분석만으로 가능. 약 1~2시간.

### 11.3 2단계: WFO 나쁜 윈도우에서 전체 조합 분포 확인

**답하려는 질문:** "WFO에서 성과가 나빴던 구간이 시장 환경의 문제인가, 아니면 IS 최적화가 OOS에서 역효과를 낸 과최적화인가?"

**필요 데이터:** ATR WFO의 각 윈도우별 전체 1,728개 조합의 OOS 성과 데이터. 만약 버퍼존 QQQ에 대해 별도의 WFO를 수행한다면, 해당 윈도우별 432개 조합의 OOS 성과 데이터가 필요하다.

**수행 방법:**

OOS 성과가 나빴던 ATR WFO Window 0, 1, 3, 5에 대해 다음을 수행한다.

1. 해당 OOS 기간에서 전체 1,728개 조합의 수익률(또는 Calmar) 분포를 히스토그램으로 그린다.
2. IS 최적으로 선택된 조합의 OOS 성과가 이 분포에서 몇 번째 백분위에 위치하는지 확인한다.

**해석 기준:**

| IS 최적의 OOS 백분위 | 해석 |
|---|---|
| 40~60% (중앙 근처) | 시장 환경의 문제. 과최적화 아님 |
| 20~40% (중앙 이하) | 경미한 과최적화 의심 |
| 0~20% (하위권) | 명확한 과최적화. IS 최적화가 역효과 |

**통과 기준:**

- 나쁜 윈도우에서 IS 최적의 백분위가 대부분 40% 이상 (시장 문제로 분류)
- 하위 20% 이하에 위치하는 윈도우가 2개 이하

**소요 시간 추정:** WFO 재실행 또는 저장된 윈도우별 결과 추출 필요. 약 2~4시간.

**참고:** 이 분석을 버퍼존 QQQ에 대해서도 수행하려면, QQQ 전략에 별도의 WFO를 구성해야 한다. 현재 QQQ 전략은 전체 기간 그리드 서치만 수행한 상태이므로, WFO를 추가 실행하는 것을 권장한다.

### 11.4 3단계: 교차 자산 검증 (SPY/IWM)

**답하려는 질문:** "버퍼존 전략이 QQQ 과거 데이터를 암기한 것인가, 아니면 범용적 추세추종인가?"

**수행 방법:**

2. **시그널 소스 교체**: 시그널 소스를 QQQ→SPY, QQQ→IWM으로 변경하고, 매매 대상도 동일하게 SPY, IWM으로 변경한다.

3. **동일한 백테스트 실행**: 슬리피지, 체결 방식 등 모든 조건을 동일하게 유지한다.

4. **결과 비교**:

   | 지표 | QQQ (기존) | SPY | IWM |
   |---|---|---|---|
   | CAGR | 10.99% | ? | ? |
   | MDD | -36.49% | ? | ? |
   | Calmar | 0.30 | ? | ? |
   | 거래수 | 14 | ? | ? |
   | 승률 | 71.43% | ? | ? |

**해석 기준:**

| SPY/IWM 결과 | 해석 |
|---|---|
| 양(+) CAGR, 양(+) Calmar | 전략의 범용성 지지. QQQ 과최적화 가능성 낮음 |
| CAGR ~0% 또는 약한 양(+) | 추세추종은 작동하나 파라미터가 QQQ에 치우침 |
| 음(-) CAGR | QQQ 데이터 암기. 과최적화 확정 |

**통과 기준:**

- SPY와 IWM 모두에서 CAGR > 0%이고 B&H 대비 Calmar가 개선됨
- 세 자산 중 2개 이상에서 양(+)의 Calmar

**소요 시간 추정:** SPY/IWM 데이터 다운로드 + 백테스트 실행. 약 2~3시간.

**추가 권장 실험 — hold_days/recent_months 제거 버전:**

3파라미터 버전(MA=200, buy_buffer=0.03, sell_buffer=0.05)도 동시에 SPY/IWM에 적용하여, 파라미터 단순화가 교차 자산 성과에 미치는 영향을 비교한다.

```
1단계 통과 + 2단계 통과 + 3단계 실패:
  → QQQ 고유의 패턴에 의존하는 전략
  → QQQ 단일 자산으로만 제한적 운용 가능 (리스크 인지 필요)
  → 파라미터 조정 없이 SPY/IWM에서 부분적 양(+)이면 제한적 유효

1단계 실패 (파라미터 불안정):
  → 전략의 견고성 부족
  → 파라미터 범위를 좁히거나 지표를 더 단순화하여 재설계
  → 또는 전략 자체를 폐기하고 새로운 가설에서 출발

2단계 실패 (IS 최적이 OOS에서 유독 나쁨):
  → 명확한 과최적화
  → 그리드 서치 기반 최적화를 포기하고, 
    경제적 논거만으로 파라미터를 사전 결정하는 접근으로 전환
```

## 12. 연구 진행 경로와 핵심 결정

> 이 절은 검증 결과를 읽기 전에 "왜 이 순서로 실험했는가"의 맥락을 제공한다.
> 개별 주제의 상세 설명은 §1~11에 있으며, 여기서는 논의가 결정으로 이어진 흐름만 기록한다.

### 12.1 연구 진행 흐름

```
1. 과최적화 학술 연구 학습 (논문 리서치)
2. 과최적화 방지 방법론 설명 (초보자용 해설 포함)
3. 전략 문서 업로드 및 분석 요청
4. 전략별 과최적화 진단 (PBO, DSR 해석)
5. WFO 해석 원칙, 경제적 논거, "이미 답을 본" 오염 문제 논의
6. 과최적화 분석 보고서 초안 생성
7. 1단계 파라미터 안정성 검증 — 구현 상세 리뷰 → 대시보드 구현 → 결과 해석
8. 판정 로직 수정 (단일 최적 vs 평균 집계 비교 문제)
9. 해석 패널 추가 프롬프트 생성
10. 본 보고서에 1단계 결과 반영 업데이트
11. 교차 자산 검증(3단계) 계획 수립
12. 교차 자산 검증 결과 분석
13. hold_days 파라미터 심층 학술 연구
14. 최종 통합 문서 생성 (현재)
```

### 12.2 초기 논의에서 확립된 핵심 원칙

아래는 연구 흐름 1~6단계에서 이루어진 학술 연구 학습과 전략 진단의 요약이다. 각 주제의 상세 설명은 §1~11에 있으며, 여기서는 **논의가 어떤 결론으로 이어졌는가**만 기록한다.

**과최적화 학술 연구 학습 (흐름 1~2):**

Bailey et al.(2014)의 "기대 SR=0인 전략도 1,000회 시행 시 최대 SR 3.26" 결과가 핵심 경고로 작용했다. 데이터 스누핑 검정의 발전(White 2000 → Hansen 2005 → Romano-Wolf 2005 → MCS 2011)을 학습하고, 실전 방지 기법(WFO, CPCV, DSR, 몬테카를로, 거래 비용 모델링)을 정리했다. → 이 학습이 이후 전략 진단의 분석 프레임워크를 결정했다.

**전략별 과최적화 진단 (흐름 3~4):**

3종 전략의 PBO/DSR을 비교한 결과, ATR WFO(PBO 0.65, DSR Z-Score -0.390)는 "운보다 못한" 상태임이 확인되었고, 전략 진화(QQQ→TQQQ→ATR)가 과최적화를 악화시키는 "복잡성의 역설"이 발견되었다. → 버퍼존 QQQ(PBO 0.40, DSR 0.78)를 검증의 기준점으로 선정하게 된 직접적 근거이다 (§2.2 참조).

**WFO 해석 원칙 확립 (흐름 5):**

"WFO에서 어떤 기간은 성과가 나쁜데, 전체기간 최적도 WFO도 둘 다 나쁘면 과최적화가 아닌 거 아닌가?"라는 질문을 계기로, WFO의 궁극적 목표를 재정의했다. WFO는 "모든 구간에서 돈 벌어야 한다"는 도구가 아니라, "IS 최적이 OOS에서 유독 나쁜가?"를 측정하는 진단 도구이다. 시장이 나빠서 전체 조합이 다 같이 안 좋은 것(시장 문제)과 IS 최적만 유독 안 좋은 것(과최적화)을 구분해야 한다. → 이 원칙이 §6의 WFO 해석 프레임워크로 반영되었다.

**경제적 논거 평가 체계 확립 (흐름 5):**

전략의 각 파라미터에 S~C 방어 등급을 부여하는 체계를 도입했다. 추세추종=S, 200일 MA=A, 버퍼 값=B, hold_days=B~C(당시). "시장 참여자들도 동일하게 생각한다"는 자기실현적 예언(긍정)과 혼잡 거래 위험(부정)의 양면성도 논의되었다. → §8의 경제적 논거 평가로 정리되었고, 이후 hold_days 학술 조사(§16)에서 B~C→S로 상향되었다.

**"이미 답을 본" 오염 문제 인식 (흐름 5~6):**

432개 그리드 서치 실행 후 그 데이터셋에 대한 순수 OOS 검증은 영원히 불가능하다는 점을 인식했다. 해결을 위한 3층 접근(①연구자가 본 적 없는 데이터로 검증, ②DSR을 N≈2,500+로 보수적 재계산, ③경제적 논거로 방어)을 수립했다. → 이 인식이 교차 자산 검증(§14~15)과 페이퍼 트레이딩 권고(§17.6)의 직접적 동기이다.

### 12.4 주요 전환점 해설

**전환점 1 — "복잡성의 역설" 발견 (초기 진단)**

PBO/DSR 진단에서 QQQ(5P) → TQQQ(5P) → ATR WFO(7P)로 전략이 진화할수록 과최적화 지표가 악화되는 패턴이 발견되었다 (§2.3 참조). 이 발견이 "가장 단순한 전략부터 검증한다"는 방향을 결정했다.

**전환점 2 — 1단계 "조건부 통과" 판정 (파라미터 안정성)**

정량적 기준 5개 중 Pass 1 / Fail 2 / Warn 2로 형식적으로는 미달이었으나, Fail 2건이 모두 sell_buffer의 체계적 방향성(넓을수록 좋음)에 기인하여 경제적으로 설명 가능했다. 이를 "조건부 통과"로 판정하고 3단계 교차 자산 검증으로 진행하는 근거가 되었다.

**전환점 3 — 3단계에서 hold_days 제거 결정과 그 부작용**

1단계에서 hold_days 0~5가 안정적이었고, 경제적 논거가 B~C등급으로 약했기에 교차 검증의 순수성을 위해 제거(hold_days=0)하였다. 그러나 결과에서 Calmar 33% 하락, MDD 12.7%p 악화가 확인되어, hold_days의 중요성을 재발견하게 되었다. 이 부작용이 §16 hold_days 심층 학술 연구의 직접적 동기이다.

**전환점 4 — DSR 처리 원칙 확립**

교차 자산 검증 계획 수립 중 "다른 ticker인데 왜 DSR을 합쳐서 통계?"라는 질문이 제기되어, "ticker별 별도 N, 별도 DSR" 원칙을 확립했다. 파라미터 고정 시 각 검증 자산의 N=1이므로 DSR 자체가 불필요하다는 결론에 도달했다 (§14.4 참조).

**전환점 5 — hold_days 방어 등급 상향**

hold_days 제거의 부작용을 분석하면서 학술 문헌을 조사한 결과, 4가지 독립적 메커니즘(단기 반전, 미시구조 노이즈, 노이즈 트레이더 과잉 반응, 가짜 돌파)이 확인 기간의 필요성을 지지하는 것을 발견했다. 이에 따라 방어 등급을 B~C에서 S(존재)/A~B(값)로 상향하고, 데이터에서 최적값을 찾는 대신 학술 근거로 hold_days=2를 사전 결정하는 접근을 권고하게 되었다 (§16 참조).

### 12.5 검증 진행 상태

```
1단계: 파라미터 안정성  ✅ 조건부 통과 (§13)
2단계: WFO 분석        ⏭️ 건너뜀 (3단계 우선 수행)
3단계: 교차 자산 검증   ✅ 통과 (§15)

다음: hold_days=2 복원 후 재검증 → 페이퍼 트레이딩 (§17)
```

## 13. 1단계 실행 결과: 파라미터 안정성 검증

> 원래 계획: §11.2에서 수립
> 실행일: 2026-03-06

### 13.1 구현 및 실행

구현 상세 문서 리뷰 후 평균 집계 방식을 채택하고, 별도 Streamlit 앱(app_parameter_stability.py)으로 대시보드를 구축하여 A~D 4개 섹션으로 시각화를 수행하였다.

실행 과정에서 판정 로직 오류를 발견하여 수정하였다: 기존에는 "단일 최적(0.301) vs 평균 집계(0.2489)"를 비교하여 Fail로 판정했으나, 이는 비교 대상이 불일치하는 문제였다. "평균 vs 평균" 비교로 수정하였다.

### 13.2 정량적 판정 결과

| 기준 | 결과 | 판정 |
|---|---|---|
| 432개 중 Calmar > 0인 조합 과반수 | 432개 전체 (100%) | **Pass** |
| 인접 파라미터 Calmar ≥ 최적의 70% | 일부 Fail | **Fail** |
| 히트맵 고원 형태 | 관찰됨 | **Pass 경향** |
| buy_buffer 축 안정성 | 차이 0.009~0.021 | **Warn** |
| sell_buffer 방향성 | 체계적 증가 | **Fail** |

정량적 종합: Pass 1 / Fail 2 / Warn 2 → **형식적 미달**

### 13.3 실질적 판정: 조건부 통과

Fail 2건은 모두 sell_buffer의 체계적 방향성에 기인한다. sell_buffer가 넓을수록 Calmar가 높아지는 패턴은 "넓은 매도 버퍼가 추세의 자연스러운 되돌림에서 조기 청산을 방지한다"는 경제적 논거와 일치한다.

핵심 증거:

- **432개 전 조합 Calmar > 0 (100%)**: 어떤 파라미터를 선택해도 양(+)의 위험 조정 수익. 전략 구조 자체의 견고성.
- **MA=200 내 buy_buffer 축 차이 0.009~0.021**: 매수 진입 시점의 미세 조정은 성과에 거의 영향 없음 (안정적 고원).
- **sell_buffer 방향 체계적 증가**: 과최적화가 아닌 경제적 논거로 설명 가능한 패턴.

이에 따라 1단계를 "조건부 통과"로 판정하고, 2·3단계 검증을 진행하는 근거로 삼았다.

| 항목 | 설명 |
|---|---|
| 파라미터 안정성 분석 대시보드 | app_parameter_stability.py — 구현 완료, 해석 패널 추가 |
| 판정 로직 수정 | 평균 대 평균 비교로 수정 완료 |
| 해석 패널 | A~D 섹션별 3계층 해석 텍스트 (용어 설명 / 해석 방법 / 현재 결과 해석) |

## 14. 3단계 실험 설계: 교차 자산 검증

> 원래 계획: §11.4에서 수립
> 상세 설계일: 2026-03-07
> 구현 참고: 부록 B 참조

### 14.1 핵심 질문

"버퍼존 전략이 QQQ 과거 데이터를 암기한 것인가, 아니면 범용적 추세추종인가?"

파라미터를 완전 고정하여 QQQ 이외의 6개 자산에 적용했을 때 양(+)의 수익이 나오는지 확인한다. 파라미터 고정 시 다중 검정 문제가 발생하지 않으므로 DSR 보정이 불필요하다.

### 14.2 고정 파라미터 (3P 버전)

| 파라미터 | 값 | 비고 |
|---|---|---|
| ma_window | 200 | EMA |
| buy_buffer_zone_pct | 0.03 (3%) | 고정 |
| sell_buffer_zone_pct | 0.05 (5%) | 고정 |
| hold_days | 0 | 제거 (기존 5P에서는 3) |
| recent_months | 0 | 제거 (동적 조정 비활성화) |

**hold_days와 recent_months 제거 근거:**

1. 1단계 검증에서 hold_days 0~5 전체가 안정적. hold_days 변경이 성과에 미치는 영향 미미.
2. 최적 파라미터가 이미 recent_months=0. 동적 조정 메커니즘 미사용이 최적이므로 제거가 자연스러움.
3. 보고서 §8.2에서 hold_days는 B~C등급(경제적 방어 거의 불가)으로 평가. 논거가 약한 파라미터 제거는 전략 방어력 향상.
4. 파라미터 3개로 줄면 "이 3개가 다른 자산에서도 작동하는가"라는 질문이 깔끔해짐.

> **후속 발견:** 이 결정으로 hold_days 제거의 부작용(Calmar 33% 하락)이 확인되었다 (§15.3 참조). 이는 §16 hold_days 심층 분석의 직접적 동기가 되었으며, hold_days 방어 등급이 S등급으로 상향되었다.

### 14.5 DSR 처리 원칙

DSR의 N은 **동일한 데이터셋에서 시도한 횟수**만 카운트한다. 다른 ticker에서의 시행은 해당 ticker의 N에 포함되지 않는다.

```
QQQ 데이터에서:
  그리드 서치 432개 + 이전 탐색 → N ≈ 2,500+
  → QQQ의 DSR 계산에만 사용

SPY 데이터에서 (파라미터 고정):
  N = 1 (시행 없음, 1개 파라미터만 적용)
  → DSR 불필요 — 선택 편향이 없으므로
```

파라미터를 고정하면 다중 검정 문제 자체가 사라진다. SPY에서 양(+) 수익이 나오면 "QQQ에서 과최적화된 파라미터인데 SPY에서도 작동" = 별도 보정 없이 유의미하다.

## 16. hold_days 파라미터 심층 분석

> 이 절은 §15.3에서 확인된 hold_days 제거의 부작용을 분석하면서 수행한 학술 조사의 결과이다.

### 16.1 파라미터 존재 자체의 정당성

hold_days(돌파 후 확인 기간)는 추세추종에서 "확인(confirmation)"이라 불리는 보편적 개념이다. "돌파 후 확인 기간을 둔다"는 개념 자체는 데이터 마이닝이 아니라 시장의 구조적 특성에 기반한다.

### 16.2 4가지 학술적 메커니즘

**메커니즘 1 — 단기 반전 효과 (Short-term Reversal):**

Lehmann(1990)은 주간 수익률 상위 종목이 다음 주에 0.35~0.55% 반전한다는 것을 문서화했다. Da, Liu & Schaumburg(2014)는 이 반전의 89%가 비펀더멘탈 요소에서 기인함을 분해하여 보여주었다. 단기 가격 움직임의 대부분은 노이즈이며, 돌파일의 극단적 수익률은 부분 반전 가능성이 가장 높다.

**메커니즘 2 — 시장 미시구조 노이즈:**

Roll(1984)의 bid-ask bounce에 의한 음의 자기상관, Aït-Sahalia & Yu(2009)의 미시구조 노이즈 63% 설명. 돌파일 종가는 장 마감 시점의 기관 물량이나 무작위 호가 위치를 반영할 수 있다. 1~3일 대기로 실제 수급 균형으로 안착.

**메커니즘 3 — 노이즈 트레이더 과잉 반응:**

De Long et al.(1990)의 긍정적 피드백 모형: 초기 움직임 → 모멘텀 트레이더 과잉 → 부분 수정 → 추세 재개. Hong & Stein(1999)의 점진적 정보 확산 모형도 동일 예측. 확인 기간은 거품 국면을 건너뛰는 역할.

**메커니즘 4 — 가짜 돌파의 높은 비율:**

Bulkowski의 13,932개 차트 패턴(1991~2008) 분석: 돌파 실패율 14~28%. 3일 연속 종가 유지 시 실패율 약 절반으로 감소, 되돌림 확률 67%→33%.

### 16.3 Jegadeesh-Titman의 스킵 기간 — 동일 개념의 월간 버전

J-T(1993)의 "2-12 모멘텀"(최근 1개월 제외)은 "단기 반전 노이즈를 건너뛰기 위한 스킵 기간"이다. hold_days는 일간 빈도의 스킵 기간으로, 동일한 학술적 논거에 기반한다.

### 16.4 최적 확인 기간에 대한 실증

**Alvarez(2023):** SPY/QQQ에서 200일 MA 돌파 후 N일 연속 유지 조건을 N=1~10 테스트(2000~2023). N=1→N=2에서 거래 수 거의 절반 감소, 수익률 0.12%p만 하락, 낙폭 의미 있게 개선. N=2부터 N=10까지 모든 값이 N=1보다 우수.

**Safari & Schmidhuber(2025):** 14년 틱 데이터 + 330년 월간 데이터에서 시간구조 확인: ~30분 미만은 평균 회귀, 1시간~2년은 추세, 2년 이상은 다시 평균 회귀. 2~5일 확인 기간은 "평균 회귀에서 추세로 전환되는 경계"에 정확히 위치.

**LeBaron(1998), Moskowitz et al.(2012):** 넓은 파라미터 범위에서 결과가 유사. "0에서 2~3으로 가는 것"이 전체 개선의 대부분.

### 16.5 전문 CTA들의 접근

**Robert Carver(前 Man AHL):** 이진적 신호 대신 -20~+20 연속적 예측값 사용. 다중 EWMA 교차 속도(2/8부터 64/256까지)를 결합하여 느린 교차가 빠른 교차를 자연스럽게 확인.

**Perry Kaufman:** 효율성 비율(ER = 순변화/절대변화합) 기반 적응적 이동평균(KAMA). ER 높으면 빠르게, 낮으면 느리게 반응하는 적응적 확인 메커니즘.

**Andreas Clenow:** 진입 확인 대신 50+ 시장에 걸친 포트폴리오 분산에 의존. "매수/매도 규칙에 집중하면 실패한다"는 입장.

### 16.6 변동성에 따른 확인 기간의 실질적 의미

같은 고정 일수라도 변동성이 다른 자산에서는 의미가 달라진다.

| 자산 | 일평균 변동성 | 3% 돌파 ÷ 변동성 | 해석 |
|---|---|---|---|
| QQQ | ~1.5% | ~2.0일치 | 적절 |
| IWM | ~1.8% | ~1.7일치 | 노이즈 가능성 높음 |
| GLD | ~0.8% | ~3.8일치 | 의미 있는 움직임 |

이론적으로 ATR 배수 기반 적응적 확인이 우월하나, 고정 2~3일도 대부분의 개선을 포착한다.

### 16.7 "0~5 범위가 결론을 제약한다"는 문제

기존 그리드 서치의 hold_days 범위 [0, 2, 3, 5]는 사용자가 임의로 설정한 값이다. "2~5일 사이는 고원"이라는 이전 결론은 이 범위 안에서만 유효하다. [0, 1, 2, 3, 5, 7, 10]으로 탐색했다면 다른 결론이 나왔을 수 있다.

따라서 "데이터에서 최적값을 찾는다"는 접근 대신 **"경제적 논거에서 값을 사전 결정한다"**는 접근이 필요하다. §16.2~16.4의 학술 근거가 이 사전 결정의 기반이다.

### 16.8 방어 등급 재평가

| 평가 항목 | 이전 등급 (§8.2) | 수정 등급 | 근거 |
|---|---|---|---|
| 파라미터 존재의 논거 | B~C | **S** | 4가지 독립적 학술 메커니즘 |
| 값 = 3의 논거 | C | **B** | 2~5일 범위가 학술적 최적이나 "정확히 3"의 특별한 이유 없음 |
| 값 = 2로 사전 결정 | — | **A** | "0→2의 개선이 최대"라는 실증 근거로 사전 결정 가능 |

> **§8.2의 원래 평가 "hold_days=3이 최적이다: B~C등급"은 이 재평가에 의해 갱신된다.** 파라미터 존재 자체는 S등급, 값은 학술 근거에 기반하여 2일로 사전 결정하는 것이 A등급이다.

## 17. hold_days 실험 계획 및 다음 단계

### 17.1 현재 상태 요약

| 항목 | 내용 |
|---|---|
| hold_days=0 (제거) | Calmar 0.200, MDD -49.18%, 승률 55.56% |
| hold_days=3 (기존 그리드 최적) | Calmar 0.300, MDD -36.49%, 승률 71.43% |
| 성과 차이 | Calmar 33% 하락, MDD 12.7%p 악화, 승률 16%p 하락 |
| 경제적 논거 | S등급 (4가지 독립적 학술 메커니즘) |
| 학술적 최적 범위 | 2~3일 (Alvarez 실증, Safari-Schmidhuber 시간구조) |

### 17.2 핵심 권고: hold_days=2로 사전 결정

**그리드 서치 대상에서 완전히 제외하고, 경제적 논거에 기반하여 2일로 고정한다.**

근거:

- Alvarez(2023) 실증: N=1→N=2에서 거래 수 절반 감소, 수익률 하락 0.12%p에 불과, 낙폭 유의미하게 개선
- 학술 문헌 일관된 결론: "0에서 2~3으로의 전환"이 전체 개선의 대부분
- "2인지 3인지 5인지"의 차이는 미미 (고원 확인됨)
- 2일 선택 이유: 가장 짧은 확인 기간으로 "추세 초기 수익 손실"을 최소화하면서 "가짜 돌파 필터링" 효과의 대부분을 포착

**왜 3이 아닌 2인가:**

| 기준 | 2일 | 3일 |
|---|---|---|
| 가짜 돌파 필터링 | 대부분 필터링됨 | 약간 더 필터링 |
| 추세 초기 수익 손실 | 최소 | 약간 더 큼 |
| 그리드 서치 결과와의 거리 | 기존 최적(3)과 1일 차이 | 기존 최적 그 자체 → "답을 본" 의심 |
| 경제적 논거의 순수성 | "학술 근거로 사전 결정" | "그리드 서치 결과를 추인한 것 아닌가?" 의심 가능 |

3을 선택하면 "결국 그리드 서치에서 나온 값을 쓰는 것 아니냐"는 의심을 받을 수 있다. 2를 선택하면 "학술 근거에서 독립적으로 결정한 값"이라는 주장이 더 깔끔하다.

### 17.3 이 결정으로 변하는 것

| 항목 | 변경 전 | 변경 후 |
|---|---|---|
| 파라미터 수 | 5개 (MA, buy, sell, hold, recent) | **4개** (MA, buy, sell, hold=2 고정) |
| 그리드 서치 조합 | 432개 | **108개** (hold_days 4개 값 제거) |
| recent_months | 0 (이미 최적이 미사용) | **0으로 고정, 조합에서 제외** → **27개** |
| DSR 계산의 N | 432+ | **27+** (크게 감소, DSR 개선 기대) |
| hold_days의 성격 | 최적화 대상 파라미터 | 경제적 논거로 사전 결정된 상수 |

### 17.4 실험 계획

**실험 1: hold_days=2 QQQ 기준선 확보 (최우선)**

| 항목 | 값 |
|---|---|
| 파라미터 | MA=200, buy=3%, sell=5%, hold=2, recent=0 |
| 자산 | QQQ |
| 기간 | 1999~2026 |
| 비교 대상 | QQQ 5P(hold=3), QQQ 3P(hold=0) |
| 통과 기준 | hold=2의 Calmar ≥ hold=3의 80% (≥0.240) |
| 소요 시간 | ~30분 |

**실험 2: hold_days=2로 교차 자산 재검증**

| 항목 | 값 |
|---|---|
| 파라미터 | MA=200, buy=3%, sell=5%, hold=2, recent=0 |
| 자산 | SPY, EFA, GLD (§15에서 명확히 통과한 자산) + IWM (가짜 돌파 필터링 효과 확인) |
| 비교 대상 | 3P(hold=0) 결과 |
| 소요 시간 | ~1시간 |

관심 사항: SPY는 hold=0에서 이미 Calmar 0.350이므로 추가 개선 여부. IWM은 가짜 돌파 26회가 hold=2로 필터링되는지. GLD는 영향 미미할 수 있음.

**실험 3: hold_days 범위 확장 탐색 (선택적)**

| hold_days | 탐색 이유 |
|---|---|
| 1 | 최소 확인 기간. 2일과의 차이 확인 |
| 7 | 1주일. 주간 주기 효과 존재 가능성 |
| 10 | 2주. 너무 긴 확인이 추세 초기 수익을 얼마나 손실시키는지 |

이 실험은 "최적값 찾기"가 아니라 "2일 근처의 고원이 실제로 존재하는지 확인"이다. 결과가 나와도 hold_days=2 결정을 변경하지 않는다.

**실험 4: 변동성 적응적 확인 (향후 검토)**

```
현재: "N일 연속 종가 > 상단밴드" (고정 일수)
대안: "종가가 상단밴드를 2×ATR 이상 초과" (변동성 적응)
또는: hold_days = round(base_days × current_ATR / median_ATR)
```

이 실험은 hold_days=2 확정 후 "추가 개선" 차원에서 검토한다. 파라미터가 추가되므로(ATR 기간, ATR 배수) 복잡성-성과 트레이드오프를 신중히 평가해야 한다. ATR WFO 전략이 PBO 0.65, DSR 0.35로 과최적화된 전례를 고려하면, 변동성 적응은 단순하게 구현해야 한다.

### 17.5 실험 1~2 후 의사결정

```
hold=2 QQQ Calmar ≥ 0.240 (hold=3의 80%):
  → hold_days=2 확정
  → 최종 파라미터: MA=200, buy=3%, sell=5%, hold=2, recent=0
  → 페이퍼 트레이딩 진행

hold=2 QQQ Calmar < 0.240:
  → hold=2와 hold=3의 차이가 "고원"인지 재검토
  → hold=3으로 선택하되 "학술 근거 + 그리드 결과 수렴"으로 정당화
  → 또는 hold=1을 테스트하여 "최소 확인"의 하한 확인
```

### 17.6 최종 파라미터 확정 후 로드맵

```
1. 파라미터 동결
   → MA=200, buy=3%, sell=5%, hold=2, recent=0
   → 이 파라미터를 일체 변경하지 않음

2. cross-asset B&H 벤치마크 추가 실행
   → SPY, IWM, EFA, EEM, GLD, TLT의 B&H 성과 확인
   → 전략 vs B&H 우위 정량화

3. 페이퍼 트레이딩 (6~12개월)
   → QQQ 시그널 기반 실시간 모니터링
   → 매매 시점, 수익률 기록
   → 유일한 순수 OOS 검증

4. 실전 투입 판단
   → 페이퍼 트레이딩 결과가 백테스트와 유사한 패턴이면 소규모 실전
   → 크게 괴리되면 전략 재검토

5. TQQQ 확장 (선택적)
   → QQQ 실전이 안정되면 매매 대상만 TQQQ로 교체
   → 시그널은 QQQ 유지
```

### B.2 통합 전략 모듈 설계

기존 `buffer_zone_tqqq.py`, `buffer_zone_qqq.py`를 하나의 config-driven 모듈로 통합하였다.

**config 구조 (개념):**

```python
@dataclass(frozen=True)
class BufferZoneConfig:
    strategy_name: str          # "buffer_zone_tqqq", "buffer_zone_spy" 등
    display_name: str           # "버퍼존 전략 (TQQQ)", "버퍼존 전략 (SPY)" 등
    signal_data_path: Path      # 시그널 소스 경로
    trade_data_path: Path       # 매매 대상 경로
    result_dir: Path            # 결과 저장 디렉토리
    grid_results_path: Path | None  # 그리드 서치 결과 (없으면 None)
    override_ma_window: int | None
    override_buy_buffer_zone_pct: float | None
    override_sell_buffer_zone_pct: float | None
    ma_type: str                # "ema" 등
```

**signal ≠ trade 지원:** `signal_data_path`와 `trade_data_path`를 분리하여 cross-asset(signal=trade)과 TQQQ(signal=QQQ, trade=TQQQ) 패턴 모두 지원.

**파라미터 결정 방식:** cross-asset은 override로 3파라미터 고정(grid_results_path=None). 기존 TQQQ/QQQ는 OVERRIDE → grid_results.csv → DEFAULT 폴백 체인 유지.
