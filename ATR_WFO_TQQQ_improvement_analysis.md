## 3. 핵심 문제 진단

### 문제 1: 잦은 매매 (Whipsaw)

- ATR WFO: **38회** 거래 vs 버퍼존 TQQQ: **15회** 거래 (2.5배)
- 횡보 구간에서 매수 후 ATR 스탑에 걸려 소손실 반복

### 문제 2: 수익 구간에서 조기 청산

- ATR WFO의 monthly_returns 분석 결과, `return_pct=0.0`인 달이 전체의 약 40%
- 버퍼존 TQQQ는 상승장에서 포지션을 길게 유지:
  - 2017년 전체 12개월 연속 보유
  - 2013~2014 연속 보유
- ATR WFO는 같은 구간에서 포지션 없음 (0.0% 월 다수)

### 문제 3: 이평선에 닿기도 전에 매도

- ATR 트레일링 스탑은 **최고종가** 기준으로 작동 (`highest_close - ATR × 3.0`)
- 이평선(MA)과 **완전히 독립적**으로 발동
- 예: 최고종가 300, ATR=10, 스탑=275인데, MA가 290이면 MA보다 훨씬 위에서 매도

### 버퍼존 TQQQ가 참고할 점

- MDD -86.96%는 치명적이지만, **상승장 보유력이 압도적** (CAGR 22.3%)
- 15건 중 10건 승리(66.7%), 승리 거래 평균 보유일 583일 (약 2.3년)
- "길게 보유하는 것 자체가 전략"이며, 이것이 CAGR 22.3%의 원천
- 대형 승리 거래: 2011→2015 (+359%), 2016→2018 (+179.8%), 2020→2022 (+228.4%)

## 4. ATR 매도 로직 코드 분석

### 매도 조건 (OR 조건)

`src/qbt/backtest/strategies/buffer_zone_helpers.py` 기준:

```
조건 1: 하단밴드 하향돌파
  - prev_close >= prev_lower_band AND close < lower_band
  - lower_band = MA × (1 - sell_buffer_zone_pct)

조건 2: ATR 트레일링 스탑
  - close < highest_close_since_entry - atr_value × atr_multiplier
  - highest_close_since_entry: 매수 후 signal_df 종가 기준 최고값
  - atr_value: QQQ 기준 ATR(14) EWM
  - atr_multiplier: 3.0 (전 윈도우 수렴)

OR 조건: 둘 중 하나라도 발동하면 다음 영업일 시가에 매도
```

### ATR 계산

```python
# True Range 계산 (QQQ signal_df 기준)
high_low = high - low
high_prev_close = (high - prev_close).abs()
low_prev_close = (low - prev_close).abs()
true_range = pd.concat([high_low, high_prev_close, low_prev_close], axis=1).max(axis=1)

# EWM(지수이동평균)으로 ATR 산출
atr = true_range.ewm(span=14, adjust=False).mean()
```

### 매수 체결 시 초기화

```python
# 매수 체결 시 highest_close를 현재 signal 종가로 초기화
highest_close_since_entry = signal_row[COL_CLOSE]
```

### 매일 갱신

```python
# 포지션 보유 중 매일 highest_close 갱신
if current_signal_close > highest_close_since_entry:
    highest_close_since_entry = current_signal_close
```

### "이평선에 닿기 전에 파는" 구조적 원인

ATR 스탑은 `highest_close - ATR × 3.0` 기준이므로, 최고가가 이평선보다 훨씬 위에 있으면 스탑 레벨도 이평선보다 위에 형성된다. 즉 **가격이 이평선까지 내려가기도 전에** ATR 스탑이 먼저 발동한다.

이는 설계 의도(급락 시 조기 청산으로 MDD 감소)이지만, 횡보/약한 조정에서도 동일하게 작동하여 whipsaw를 유발한다.

## 5. 개선 방안 9가지

미래 데이터 참조 금지, 과최적화 금지 원칙을 준수하는 방안만 제시한다.

### 방안 A: ATR 트레일링 스탑에 "최소 보유 기간" 추가

**핵심 아이디어:** 매수 후 N일간은 ATR 스탑을 비활성화

**현재 문제:** 매수 직후 작은 하락에도 ATR 스탑이 발동 → 소손실 누적

**개선 방법:**

- 매수 체결 후 최소 `min_hold_days` (예: 10~20 영업일)간은 ATR 스탑을 무시
- 하단밴드 매도 신호는 유지 (극단적 하락 방어)
- 이후부터 ATR 스탑 활성화

**초보자 해설:**
주식을 사자마자 바로 파는 건 "조급함"에 해당합니다. 마치 씨앗을 심고 싹이 나기도 전에 파내보는 것과 같아요. 일정 기간은 추세가 발전할 시간을 줘야 합니다. 이 방법은 "최소한 2~4주는 기다려보자"라는 규칙입니다.

- **장점:** 구현 간단, 파라미터 1개 추가, 횡보 구간 whipsaw 감소
- **단점:** 급락 시 보호 약화 가능, min_hold_days 선택에 주의 필요
- **과최적화 위험:** 낮음 (범용 파라미터, WFO에서 최적화 불필요)

---

### 방안 B: ATR 멀티플라이어를 시간 경과에 따라 점진적으로 좁히기 (Ratchet Stop)

**핵심 아이디어:** 보유 초기에는 느슨한 스탑, 시간이 지날수록 점진적으로 빡빡하게

**현재 문제:** ATR × 3.0이 항상 고정 → 수익 구간에서도 넓은 스탑, 횡보에서도 넓은 스탑

**개선 방법:**

```
보유 초기 (0~20일):  multiplier = 3.5 (넓게, 추세 발전 대기)
보유 중기 (21~60일): multiplier = 3.0 (현재와 동일)
보유 후기 (61일+):   multiplier = 2.5 (수익 보호 강화)
```

**초보자 해설:**
낚시를 생각해보세요. 처음에는 줄을 느슨하게 잡고 물고기가 입질할 시간을 줍니다. 하지만 점점 줄을 당겨서 놓치지 않게 합니다. 이 전략도 마찬가지로, 초기에는 가격이 흔들릴 여유를 주고, 이미 수익이 쌓이면 스탑을 좁혀서 수익을 지킵니다.

- **장점:** 보유 초기 whipsaw 감소 + 후기 수익 보호
- **단점:** 구간 설정 자체가 파라미터 → 과최적화 유의
- **과최적화 위험:** 중간 (구간/멀티플라이어 조합이 늘어남)

---

### 방안 C: ATR 스탑 기준가를 "최고종가"에서 "이동평균" 기반으로 변경

**핵심 아이디어:** `highest_close - ATR × mult` 대신 `MA - ATR × mult`를 스탑으로 사용

**현재 문제:** 최고종가 기준이라 작은 고점 갱신 후 되돌림에서도 스탑 발동

**개선 방법:**

```
현재: stop = highest_close_since_entry - ATR × 3.0
변경: stop = MA_200 - ATR × mult

이평선은 천천히 움직이므로 스탑이 더 안정적.
이평선 아래로 내려가야 매도 → 이평선 근처에서 팔리는 문제 해결
```

**초보자 해설:**
현재 방식은 "가장 높았던 가격에서 얼마나 떨어졌나"를 봅니다. 그런데 주가가 잠깐 높이 올랐다가 내려와도 스탑이 걸립니다. 이평선 기준으로 바꾸면, "추세의 중심에서 얼마나 떨어졌나"를 보는 겁니다. 추세 자체가 무너져야 매도하므로 횡보에서 덜 흔들립니다.

- **장점:** 이평선 안 닿고 파는 문제 원천 해결, 추세 추종 철학에 부합
- **단점:** 급락 시 이평선이 반응 느려 MDD 증가 가능
- **과최적화 위험:** 낮음 (기존 MA 200 재사용)

---

### 방안 D: 변동성 필터 (횡보 구간 진입 억제)

**핵심 아이디어:** ATR이 낮은(횡보) 구간에서는 매수 진입 자체를 차단

**현재 문제:** 횡보 구간에서도 상단밴드 돌파하면 진입 → 곧바로 ATR 스탑으로 청산

**개선 방법:**

```python
# 매수 조건에 변동성 필터 추가
atr_threshold = MA × 기준비율  # 예: MA의 1% 이상이어야 진입
if atr_value / current_close < atr_threshold_pct:
    매수 신호 무시  # 변동성이 너무 낮으면 횡보로 판단
```

**초보자 해설:**
바다가 잔잔할 때(횡보)는 파도타기를 할 수 없습니다. 파도가 어느 정도 있어야(변동성이 충분) 서핑(추세 추종)이 가능합니다. 이 필터는 "파도가 충분히 클 때만 서핑보드를 타자"는 규칙입니다.

- **장점:** 횡보 구간 whipsaw 원천 차단
- **단점:** 강한 추세 시작의 초기(아직 변동성 낮을 때) 놓칠 수 있음
- **과최적화 위험:** 낮음 (ATR 정규화 = ATR/Close는 범용적)

---

### 방안 E: 분할 매도 (Scaled Exit)

**핵심 아이디어:** 전량 청산 대신 단계별 매도

**현재 문제:** ATR 스탑 발동 = 전량 매도 → 이후 재상승하면 포지션 0

**개선 방법:**

```
1차 ATR 스탑 발동: 포지션의 50% 매도 (리스크 감소)
2차 하단밴드 돌파: 나머지 50% 매도 (완전 청산)

또는:
1차 ATR 스탑 발동: 50% 매도
추가 하락 (ATR × 4.0): 나머지 50% 매도
반등 시: 50% 포지션으로 상승 추세 참여 가능
```

**초보자 해설:**
계란을 한 바구니에 담지 말라는 원칙의 시간 버전입니다. 한 번에 다 팔면 맞으면 좋지만, 틀리면 재진입 비용이 큽니다. 반씩 나눠 팔면, 첫 매도가 틀려도(반등하면) 남은 50%로 수익을 얻고, 맞으면(더 하락) 이미 절반은 지킨 겁니다.

- **장점:** 상승장에서 최소 50% 포지션 유지, whipsaw 영향 반감
- **단점:** 구현 복잡도 증가 (현재 전량진입/전량청산 구조 변경 필요)
- **과최적화 위험:** 낮음 (50/50 분할은 범용적)

---

### 방안 F: 추세 강도 지표(ADX) 활용한 조건부 스탑

**핵심 아이디어:** 추세가 강할 때는 스탑을 넓게, 추세가 약할 때는 스탑을 좁게

**현재 문제:** ATR × 3.0이 시장 상태와 무관하게 고정

**개선 방법:**

```
ADX > 25 (강한 추세): multiplier = 3.5 (추세 유지 우선)
ADX < 20 (추세 약함/횡보): multiplier = 2.5 (빠른 청산)

ADX(Average Directional Index): 추세의 강도를 0~100으로 측정 (방향 무관)
일반적으로 ADX > 25이면 추세, < 20이면 횡보
```

**초보자 해설:**
ADX는 "지금 시장에 방향성이 있는가?"를 알려주는 나침반입니다. 나침반이 강하게 한 방향을 가리키면(ADX 높음) 그 방향을 따라가야 하고, 빙빙 돌면(ADX 낮음) 멈추는 게 낫습니다. 추세가 강할 때는 느슨하게 보유하고, 추세가 약할 때는 빨리 정리하는 겁니다.

- **장점:** 시장 상태에 적응하는 동적 스탑, 학술적 근거 있음
- **단점:** ADX 파라미터(기간, 임계값) 추가 → 탐색 공간 증가
- **과최적화 위험:** 중간 (ADX 기간은 14일 범용값 사용 시 감소)

### 방안 G: 재진입 쿨다운 (Re-entry Cooldown)

**핵심 아이디어:** ATR 스탑 청산 후 일정 기간 재진입 금지

**현재 문제:** ATR 스탑 청산 → 바로 상단밴드 돌파 → 재진입 → 또 ATR 스탑 (반복)

**개선 방법:**

```
ATR 스탑으로 매도한 후 N일간 매수 신호 무시
(하단밴드 매도는 쿨다운 없음 - 이건 진짜 추세 전환)
```

**초보자 해설:**
추세 추종에서 반복 손절은 거래 비용을 갉아먹습니다. ATR 스탑은 "일시적 위험 회피"인데, 바로 다시 들어가면 같은 위험에 재노출됩니다. 마치 뜨거운 냄비를 만져서 손을 뗐는데 바로 다시 만지는 것과 같습니다. 잠깐 기다리면 상황이 정리됩니다.

- **장점:** whipsaw 비용 직접 감소, 구현 간단
- **단점:** 진짜 재진입 기회를 놓칠 수 있음
- **과최적화 위험:** 낮음 (5~10일 범용값)

---

### 방안 H: 버퍼존 매도를 "N일 연속 하단밴드 하회"로 강화

**핵심 아이디어:** 하단밴드 1일 하회가 아닌 N일 연속 하회 시에만 매도

**현재 문제:** 하단밴드를 1일만 살짝 하회하고 반등하는 경우도 매도 발생

**개선 방법:**

```
현재: 전일 >= lower_band AND 당일 < lower_band → 즉시 매도
변경: N일 연속 lower_band 하회 시에만 매도 확정
      (매수 시 hold_days와 대칭적 개념)
```

**초보자 해설:**
현재는 매수할 때는 hold_days로 "며칠 확인하고" 사는데, 매도할 때는 1일 만에 팝니다. 비대칭적이죠. 팔 때도 "진짜 하락인지 며칠 확인하고" 팔면, 일시적 하락에 흔들리지 않습니다. 매수에 신중한 것처럼 매도에도 신중해지는 겁니다.

- **장점:** 매수/매도 대칭성 확보, 일시적 급락 후 반등에서 보유 유지
- **단점:** 진짜 하락 시 청산 지연 → MDD 증가 가능
- **과최적화 위험:** 낮음 (1~3일 범용값)

---

### 방안 I: 이중 이동평균 체계 (Dual MA Filter)

**핵심 아이디어:** 장기 MA(200일) + 중기 MA(50일)를 함께 활용

**현재 문제:** 200일 MA 하나로 진입/청산 모두 판단 → 횡보 구간에서 애매한 신호

**개선 방법:**

```
진입 조건: 종가 > MA200 × (1 + buy_buffer) AND MA50 > MA200 (골든크로스 상태)
청산 조건: 기존 유지 (하단밴드 OR ATR 스탑)

MA50 > MA200: 중기 추세도 상승 확인 → 횡보 구간에서 가짜 돌파 필터링
```

**초보자 해설:**
200일 이평선만 보면 "큰 추세"만 알 수 있습니다. 50일 이평선은 "최근 추세"를 보여줍니다. 두 이평선이 모두 상승 방향일 때(골든크로스)만 매수하면, 큰 추세와 최근 추세가 일치하는 확실한 상승 구간에서만 진입합니다. 이중 확인(double confirmation)이죠.

- **장점:** 학술적으로 검증된 방법, 횡보 필터 효과
- **단점:** 진입 지연 가능 (MA50이 MA200 위로 올라와야 함)
- **과최적화 위험:** 낮음 (50/200은 가장 범용적인 조합)

## 7. 추천 조합

단일 방안보다 상호 보완적인 조합이 효과적이다. 과최적화 위험이 낮은 것 위주로:

### 1순위 조합: A + D + G

- **최소 보유기간 (A):** 진입 직후 whipsaw 방지
- **변동성 필터 (D):** 횡보 구간 진입 자체를 차단
- **재진입 쿨다운 (G):** 반복 손절 방지
- 추가 파라미터: 3개 (`min_hold_days`, `atr_threshold_pct`, `cooldown_days`)
- 모두 범용값 설정 가능 → 과최적화 위험 최소

### 2순위 조합: C + H

- **MA 기준 스탑 (C):** 이평선 안 닿고 파는 문제 근본 해결
- **매도 확인일 (H):** 일시적 하락 필터링
- 구조적 변경이지만 파라미터 증가 최소

### 3순위 (대폭 개편): E + I

- **분할 매도 (E):** 전량진입/전량청산 구조 자체를 변경
- **이중 MA (I):** 진입 품질 향상
- 구현 난이도 높지만, 버퍼존 TQQQ의 "장기 보유력"에 가장 가까워짐

## 11. Grid 분석에서 도출된 시사점

### 시사점 1: sell_buffer=0.05가 최적인 이유

TQQQ의 일일 변동성이 QQQ의 3배이기 때문.

- `sell=0.03`이면 QQQ 기준 3% 하락에서 매도 → TQQQ 환산 약 9%의 자본 손실
- `sell=0.05`면 QQQ 5% 하락 = TQQQ 약 15% → 더 넓은 여유로 노이즈와 진짜 하락 구분

ATR WFO Dynamic에서 sell_buffer 변화:

- W1~W5: `sell=0.03` (Period 1, 좁음)
- W7~W8: `sell=0.05` (Period 2, 넓음)
- W9~W10: `sell=0.03` (최근, 다시 좁아짐)

W9~W10에서 `sell=0.03`으로 돌아간 것은, ATR 스탑이 주도적 매도를 담당하면서 sell_buffer의 역할이 축소된 결과. 하지만 Grid 분석에 따르면, **ATR 없는 환경에서는 `sell=0.05`가 확실히 우위**.

### 시사점 2: "적게 거래할수록 좋다"의 정량적 증거

### 시사점 3: 최적 파라미터 조합의 공통점

## 12. 보강된 우선순위

Grid 분석 결과를 반영하여 우선순위를 재조정한다.

### 최우선: 방안 A (최소 보유기간) + 방안 G (재진입 쿨다운)

**근거:** ATR WFO trades.csv에서 보유일 28일 미만 거래가 12건이고, 이 중 11건이 손실. 최소 30 영업일(약 6주) 보유 규칙만으로도 이 11건의 소손실을 방지 가능. Grid에서 상위 조합의 평균 보유 패턴과도 일치.

- **예상 효과:** 거래수 38 → 약 25회 이하로 감소
- **예상 MDD:** 일부 급락 시 보호 약화되나, Grid 증거상 거래수 감소의 효과가 더 큼

### 2순위: 방안 D (변동성 필터)

**근거:** W3(2011~2013, CAGR -20.88%), W5(2015~2017, CAGR -17.46%)는 횡보+변동성 구간에서 9회, 8회 거래하며 큰 손실. ATR 정규화 필터로 이 구간의 진입 자체를 차단하면 거래수 추가 감소.

### 3순위: 방안 C (MA 기준 스탑) or 방안 H (매도 확인일)

**근거:** Grid에서 `sell=0.05`의 우위가 증명됨. 현재 ATR 스탑이 sell_buffer보다 먼저 발동하여 `sell=0.05`의 인내심을 무력화. MA 기준 스탑으로 변경하면 "이평선 아래로 내려가야 매도"가 되어 Grid의 `sell=0.05` 효과와 유사하게 작동.
