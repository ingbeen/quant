# WFO 정책 및 전략 생명주기 관리 기준

> 작성일: 2026-03-14
> 기반 분석: overfitting_analysis_report.md, app_walkforward.py 대시보드 결과
> 목적: WFO 동적 변경을 사용하지 않는 이유, WFO 점검 주기, 전략 폐기·재설계 기준을 명시

---

## 1. WFO 동적 파라미터 변경을 기준으로 하면 안 되는 이유

### 1.1 이 프로젝트에서 직접 증명된 증거 3가지

**증거 1 — ATR WFO 전략의 실패:**

WFO 동적 파라미터 변경을 적용한 ATR WFO(TQQQ) 전략이 고정 파라미터 전략보다 과최적화 지표가 모두 나빴다.

| 전략 | 파라미터 변경 방식 | PBO | DSR | DSR Z-Score |
|---|---|---|---|---|
| 버퍼존 QQQ | **고정** | 0.40 | 0.78 | +0.771 |
| ATR WFO TQQQ | **WFO 동적** | 0.65 | 0.35 | **-0.390** |

ATR WFO의 DSR Z-Score가 음수(-0.390)라는 것은, 1,728개 조합을 탐색한 결과의 최고 SR이 순전히 운으로 달성 가능한 기대 최대 SR보다도 낮다는 뜻이다. 파라미터를 동적으로 바꾼 전략이 운보다 못했다.

**증거 2 — QQQ WFO Dynamic = Fixed 성과 동일:**

WFO 대시보드에서 QQQ의 결과:

```
Dynamic (매 윈도우 재최적화): Stitched CAGR 11.7%
Fixed (첫 파라미터 고정):     Stitched CAGR 11.7%
```

매 윈도우마다 최적 파라미터를 새로 찾아서 적용해도, 처음 값을 고정해도 성과가 동일하다. 동적 변경이 가치를 더하지 않는다.

**증거 3 — TQQQ Dynamic의 수익 집중 문제:**

```
TQQQ Dynamic: CAGR 21.4%  (높아 보임)
TQQQ Fixed:   CAGR  7.0%  (낮음)
```

Dynamic이 3배 좋아 보이지만, PC 최대(Profit Concentration)가 0.70이다. 전체 수익의 70%가 단 하나의 구간(2009년 급반등)에서 나왔다. 이것은 "적응이 잘 작동한 것"이 아니라 "운 좋게 한 구간에서 대박이 난 것"이며, 재현 가능성이 낮다.

### 1.2 학술적 근거

**탐색 횟수의 곱셈 효과:**

WFO를 실행하면 매 윈도우마다 그리드 서치를 수행한다. 실질적 시행 횟수는:

```
고정 파라미터:     N = 432 (1회 그리드 서치)
WFO 동적 (11윈도우): N = 432 × 11 = 4,752
WFO 동적 (22윈도우): N = 432 × 22 = 9,504
```

Bailey et al.(2014)에 따르면 기대 최대 SR ≈ √(2 × ln(N))이므로, N이 커질수록 "운으로 좋은 결과가 나올 확률"이 높아진다. WFO 동적 변경은 이 N을 윈도우 수만큼 곱하여 과최적화 위험을 기하급수적으로 증가시킨다.

**"적응"이 아니라 "노이즈 추종":**

"시장이 바뀌니까 파라미터도 바꿔야 한다"는 논리의 숨겨진 가정은 "IS에서 찾은 최적이 다음 OOS에서도 최적일 것"이다. 이것이 성립하려면 시장 변화가 점진적이고 예측 가능해야 한다. 그러나 실제 시장의 regime change(2008 금융위기, 2020 코로나, 2022 금리 인상)는 급격하고 예측 불가능하다.

결과적으로 WFO 동적 변경이 실제로 하는 것은 "직전 과거의 노이즈에 맞춘 파라미터를 다음 구간에 적용"하는 것이다.

### 1.3 고원이 "바꾸지 않아도 된다"는 구조적 증거

182회 고원 분석(§17~18)에서 4개 핵심 파라미터 모두 고원 안에 위치함이 확인되었다.

| 파라미터 | 고원 범위 | 현재 값 | 고원 내 Calmar 변동 |
|---|---|---|---|
| ma_window | 150~300 | 200 | 0.21~0.30 |
| buy_buffer | 0.01~0.05 | 0.03 | 0.16~0.30 |
| sell_buffer | 0.03~0.07 | 0.05 | 0.22~0.30 |
| hold_days | 3~5 | 3 | 0.26~0.30 |

고원 안에서 WFO 최적이 이리저리 움직이는 것(예: buy_buffer가 1%→3%)은 **"시장이 바뀌었다"는 신호가 아니라 "고원 위의 노이즈"**이다. 성과가 비슷하므로 바꿀 이유가 없다.

## 2. WFO 점검 주기 및 방법

### 2.1 원칙: 파라미터를 바꾸기 위해서가 아니라, 전략이 여전히 작동하는지 확인하기 위해 점검한다.

WFO 점검의 목적은 "더 나은 파라미터 찾기"가 아니다. "현재 고정 파라미터가 여전히 고원 안에 있는가?"를 확인하는 진단이다.

### 2.2 점검 주기: 연 1회

| 항목 | 내용 |
|---|---|
| 주기 | **연 1회** (매년 1월 또는 데이터 갱신 후) |
| 방법 | run_walkforward.py 재실행 (최신 데이터 포함) |
| 확인 사항 | 아래 §2.3의 3가지 체크리스트 |
| 파라미터 변경 | **하지 않는다** (점검 결과에 관계없이) |

**왜 연 1회인가:**

- 이 전략의 평균 거래 빈도가 연 0.5~1회(QQQ 14거래 / 27년)이므로, 6개월 이하 주기로 점검해도 새로운 정보가 없다.
- WFO 윈도우(Expanding Window)가 2년 OOS이므로, 1년 단위 점검이 자연스럽다.
- 더 자주 점검하면 "결과를 보고 바꾸고 싶은 유혹"이 커지고, 이것이 과최적화의 시작이다.

### 2.3 연 1회 점검 체크리스트

**체크 1 — Dynamic과 Fixed의 성과 괴리:**

```
괴리 없음 (현재 상태):
  Dynamic CAGR ≈ Fixed CAGR
  → 정상. 고정 파라미터 유지.

괴리 발생:
  Dynamic CAGR >> Fixed CAGR (예: 15% vs 8%)
  → 경고. 시장 구조가 바뀌어 고정 파라미터가 부적합해지고 있을 가능성.
  → 그러나 파라미터를 바꾸지 않는다. §3(전략 폐기 기준) 참조.
```

**체크 2 — 파라미터 추이의 고원 이탈:**

```
고원 안에서 변동 (현재 상태):
  WFO 최적 buy_buffer가 1% → 3% → 2% 등으로 변동
  → 정상. 고원 내 노이즈.

고원 밖으로 이탈:
  WFO 최적 buy_buffer가 0.10 이상으로 튀거나
  WFO 최적 ma_window가 50 이하로 떨어짐
  → 심각한 경고. 전략 구조 자체의 유효성 재검토 필요.
```

**체크 3 — Stitched Equity의 장기 추세:**

```
우상향 유지:
  Stitched Equity가 장기적으로 상승 추세
  → 정상.

횡보 또는 하락:
  최근 3~5년간 Stitched Equity가 횡보 또는 하락
  → 경고. §3(전략 폐기 기준)의 정량적 기준과 대조.
```

### 2.4 점검 결과에 따른 행동

| 체크 결과 | 행동 | 절대 하지 않는 것 |
|---|---|---|
| 3개 모두 정상 | 파라미터 유지. 다음 해 재점검 | 파라미터 변경 |
| 1개 경고 | 기록만 남기고 관찰. 다음 해 재점검 | 파라미터 변경 |
| 2개 이상 경고 | §3(전략 폐기 기준) 검토 | 파라미터 미세 조정 |
| 고원 이탈 | §3.3(전략 재설계 기준) 검토 | WFO 최적으로 파라미터 교체 |

## 3. 전략 폐기 및 재설계 기준

### 3.1 원칙: 파라미터 조정이 아니라 전략 구조의 유효성을 판단한다.

"파라미터를 바꾸면 나아질까?"라는 질문은 하지 않는다. 고원 분석(§17~18)에서 고원 내 어떤 값을 골라도 비슷하다는 것이 확인되었으므로, 성과 악화의 원인은 파라미터가 아니라 시장 구조 변화이다. 파라미터 조정으로 해결할 수 있는 문제가 아니다.

### 3.2 전략 폐기 기준

아래 조건 중 **2개 이상** 충족 시 전략 폐기를 검토한다.

| # | 기준 | 임계값 | 근거 |
|---|---|---|---|
| 1 | **실전 Calmar가 B&H Calmar 이하로 3년 연속** | 전략 Calmar < B&H Calmar × 3년 | B&H보다 못하면 전략의 존재 이유가 없다. 3년은 추세추종의 구조적 횡보 기간(2~3년)을 감안한 최소 관찰 기간 |
| 2 | **실전 MDD가 백테스트 MDD의 1.5배 초과** | 예: QQQ 실전 MDD > -36.49% × 1.5 = -54.7% | 백테스트에서 예상하지 못한 수준의 손실은 모델 가정 자체가 틀렸음을 시사 |
| 3 | **WFO Dynamic과 Fixed의 성과 괴리가 3년 연속 확대** | Dynamic Calmar > Fixed Calmar × 2.0 (3년 연속) | 시장이 고정 파라미터로는 대응할 수 없는 수준으로 변했음을 시사 |
| 4 | **WFO 최적 파라미터가 고원 밖으로 3회 연속 이탈** | 4개 핵심 파라미터 중 2개 이상이 고원 밖 | 시장 구조가 근본적으로 변하여 기존 고원이 더 이상 유효하지 않음 |

**"3년"과 "3회"의 이유:** 추세추종 전략은 구조적으로 횡보장에서 손실을 보므로, 1~2년의 부진은 정상 범위이다. 3년 연속 부진은 "횡보장 때문"이 아니라 "전략이 작동하지 않음"일 가능성이 높아지는 시점이다.

### 3.3 전략 재설계 기준

폐기 기준에 도달했을 때의 행동:

```
단계 1: 전략을 중단한다 (포지션 청산)
단계 2: 원인을 분석한다
  → "모든 추세추종이 안 되는 시장인가?" (예: VIX 장기 저변동성)
  → "200일 MA 기반 추세추종만 안 되는 것인가?"
  → "우리 전략의 특수한 문제인가?"
단계 3: 원인에 따라 결정한다
  → 모든 추세추종이 안 됨: 전략 폐기, 다른 전략 유형 탐색
  → MA 기반만 안 됨: 전략 구조 재설계 (새로운 가설에서 출발)
  → 특수한 문제: 해당 문제만 수정 (예: 슬리피지 모델 갱신)
```

**재설계 시 주의:** 새 전략을 설계할 때 기존 데이터를 사용하면 "이미 답을 본" 오염이 재발한다. 가능하면 새로운 자산이나 새로운 기간의 데이터에서 출발해야 한다.

### 3.4 전략 폐기가 아닌 것

| 상황 | 행동 | 이유 |
|---|---|---|
| 1년간 수익이 0% | 관찰 지속 | 추세추종의 정상적 횡보기 |
| MDD가 백테스트 수준(-36%) 발생 | 관찰 지속 | 백테스트에서 이미 예상된 시나리오 |
| 단일 거래에서 -15% 손실 | 관찰 지속 | 거래 내역에 유사 규모 손실 존재 |
| "더 좋은 파라미터가 있을 것 같다" | **아무것도 하지 않음** | 이것이 과최적화의 시작 |

```

│  MA=200, buy=3%, sell=5%, hold=3                 │
│  4개 파라미터 전부 고원 안 확인 (182회 백테스트)     │
└──────────────────┬──────────────────────────────┘
                   ↓
┌─────────────────────────────────────────────────┐
│  실전 운용 (파라미터 동결)                          │
│  QQQ 시그널 기반 매매                              │
│  SPY 병행 모니터링                                 │
│  파라미터 변경 금지                                │
└──────────────────┬──────────────────────────────┘
                   ↓  (매년 1월)
┌─────────────────────────────────────────────────┐
│  연 1회 WFO 점검                                  │
│  체크 1: Dynamic vs Fixed 괴리                    │
│  체크 2: 파라미터 추이 고원 이탈                     │
│  체크 3: Stitched Equity 장기 추세                 │
│                                                   │
│  → 정상: 파라미터 유지, 다음 해 재점검              │
│  → 경고 1개: 기록 후 관찰                          │
│  → 경고 2개+: 폐기 기준 검토                       │
```
