# 파라미터 고원 분석 — sell_buffer / buy_buffer / ma_window 그리드 서치 실행 계획

> 작성일: 2026-03-09
> 목적: 코딩 AI에 전달하여 3개 파라미터의 고원 형태를 확인하는 그리드 서치를 구현·실행하도록 한다.
> 전제: CLAUDE.md(backtest 도메인)를 숙지한 상태에서 진행한다.
> 선행 실험: hold_days 고원 분석(56회)이 동일한 방법론으로 성공적으로 완료되었다. 이번 실험은 그 방법론을 나머지 3개 파라미터에 적용하는 것이다.

---

## 1. 배경과 목적

### 1.1 왜 이 실험이 필요한가

버퍼존 전략의 현재 파라미터(MA=200, buy=3%, sell=5%, hold=3)는 QQQ 그리드 서치(432개 조합)에서 찾은 값이다. hold_days에 대해서는 0~10 범위의 고원 분석(56회 백테스트)을 통해 "봉우리가 아닌 고원의 일부"임을 확인했다. 그러나 나머지 3개 파라미터에 대해서는 동일한 검증이 아직 수행되지 않았다.

특히 sell_buffer에서 미해결 문제가 있다. 1단계 파라미터 안정성 검증(432개 조합)에서 sell_buffer는 "넓을수록 Calmar가 높아지는 체계적 방향성"이 관찰되었다. 그러나 기존 탐색 범위가 [0.01, 0.03, 0.05]로 제한되어 있어, 현재 값 0.05가 "고원의 일부"인지 "탐색 범위의 우측 끝에 불과한 것"인지 알 수 없다.

이 문제는 hold_days에서 "기존 범위 [0, 2, 3, 5]가 결론을 제약한다"와 정확히 동일한 구조이며, 해결 방법도 동일하다: 범위를 확장하여 곡선의 전체 형태를 확인한다.

### 1.2 실험의 목적

3개 파라미터(sell_buffer, buy_buffer, ma_window) 각각에 대해, 해당 파라미터만 변경하고 나머지를 고정한 상태에서 Calmar 곡선의 형태를 확인한다.

이 실험은 **"최적값 찾기"가 아니라 "고원 형태 확인"**이다. 결과에서 Calmar 1위를 선택하는 것이 아니라, 현재 값이 고원 안에 위치하는지를 판단하는 데 사용한다. 결과에 관계없이 현재 파라미터 값을 변경하지 않는다.

### 1.3 기대 결과물

3개 실험(A, B, C) 각각에 대해 hold_days 고원 분석과 동일한 형식의 비교 테이블을 생성한다.

```
실험 A 예시 (sell_buffer):
           sell=0.01  sell=0.03  sell=0.05  sell=0.07  sell=0.10  sell=0.15
QQQ        ?          ?          0.300      ?          ?          ?
SPY        ?          ?          ?          ?          ?          ?
IWM        ?          ?          ?          ?          ?          ?
...

실험 B 예시 (buy_buffer):
           buy=0.01   buy=0.02   buy=0.03   buy=0.05   buy=0.07   buy=0.10
QQQ        ?          ?          0.300      ?          ?          ?
SPY        ?          ?          ?          ?          ?          ?
...

실험 C 예시 (ma_window):
           ma=50      ma=100     ma=150     ma=200     ma=250     ma=300
QQQ        ?          ?          ?          0.300      ?          ?
SPY        ?          ?          ?          ?          ?          ?
...
```

## 3. 실험 A: sell_buffer 고원 확인

### 3.1 고정 파라미터

| 파라미터 | 값 | 비고 |
|---|---|---|
| ma_window | 200 | EMA, **고정** |
| ma_type | ema | **고정** |
| buy_buffer_zone_pct | 0.03 (3%) | **고정** |
| hold_days | 3 | **고정** |
| recent_months | 0 | **고정** |

### 3.2 탐색 값

```
sell_buffer_zone_pct = [0.01, 0.03, 0.05, 0.07, 0.10, 0.15]
```

- 0.01, 0.03, 0.05: 기존 그리드 범위. 1단계 결과와의 일관성 확인.
- 0.07, 0.10: 기존 범위 밖. "넓을수록 좋다"가 어디서 꺾이는지 확인하는 핵심 구간.
- 0.15: 극단값. 15% 매도 버퍼는 QQQ 변동성(~1.5%) 기준 약 10일치 움직임. 고원의 우측 경계를 확실하게 포착하기 위한 값.

### 3.3 실행 횟수

6값 × 7자산 = **42회** 백테스트

### 3.4 검증 포인트

QQQ sell=0.05의 Calmar이 기존 결과(0.300)와 일치하는지 확인한다. 일치하지 않으면 고정 파라미터 설정에 오류가 있는 것이다.

## 8. 제약사항

### 8.1 절대 제약

1. **각 실험에서 해당 파라미터 이외의 파라미터를 변경해서는 안 된다.** 실험 A에서는 sell_buffer만 변경하고 ma_window, buy_buffer, hold_days는 반드시 고정값을 유지한다. 실험 B, C도 마찬가지.

2. **결과에서 "최적값"을 선택하는 로직을 넣지 않는다.** 이 실험의 목적은 최적화가 아니라 형태 확인이다.

3. **기존 전략의 결과 파일을 덮어쓰지 않는다.** 이번 실험의 결과는 별도 경로에 저장한다.

### 8.2 결과 저장 원칙

- 126회 각각의 성과 지표가 보존되어야 한다.
- 최소한 실험명, 파라미터명, 파라미터값, 자산명, CAGR, MDD, Calmar, 거래수, 승률이 기록되어야 한다.
- 피벗 테이블은 터미널에 출력하고, 동시에 CSV 파일로도 저장한다.

## 9. 구현 방향성

### 9.1 핵심 구조

hold_days 고원 분석 스크립트와 동일한 구조를 사용한다. 기존 버퍼존 전략 백테스트 엔진을 그대로 사용하되, 변경 대상 파라미터만 외부에서 주입하여 반복 실행하는 래퍼 스크립트이다.

### 9.2 실행 흐름

```
실험 A, B, C 각각에 대해:
  1. 탐색 값 목록 × 7개 자산 조합 생성
  2. 각 조합에 대해:
     a. 고정 파라미터 + 해당 탐색 값으로 백테스트 실행
     b. 결과(CAGR, MDD, Calmar, 거래수, 승률) 수집
  3. 피벗 테이블로 정리
  4. 터미널 출력 + CSV 저장

전체 완료 후:
  5. 통합 상세 테이블 생성
```

### 9.3 기존 코드 활용

- 이 프로젝트에는 이미 버퍼존 전략의 백테스트 엔진이 구현되어 있다.
- 기존 `BufferZoneConfig` 기반 통합 전략 모듈과 `run_single_backtest.py` 스크립트가 있다.
- hold_days 고원 분석 스크립트가 이미 구현되어 있으므로, 그 구조를 참고하면 된다.
- CLAUDE.md(backtest 도메인)에 전략 모듈 구조, config 패턴, 파라미터 오버라이드 방식이 설명되어 있으므로 반드시 먼저 읽고 기존 구조를 파악한 뒤 구현한다.

### 9.4 구현 시 주의사항

- 기존 전략 결과 디렉토리(`storage/results/backtest/buffer_zone_*`)를 건드리지 않는다. 이번 실험 결과는 별도 경로(예: `storage/results/backtest/param_plateau/`)에 저장한다.
- hold_days 고원 분석 스크립트의 결과 파일도 건드리지 않는다.
- 각 자산의 데이터 파일 경로는 `common_constants.py`에 이미 정의되어 있으므로 먼저 확인한다.

## 10. 실행 후 확인 사항

### 10.1 검증 포인트

3개 실험 모두에서, 현재 확정값에 해당하는 조합의 Calmar이 기존 결과와 일치하는지 확인한다.

| 확인 항목 | 기대값 | 해당 실험 |
|---|---|---|
| QQQ, sell=0.05 (나머지 고정) Calmar | 0.300 | 실험 A |
| QQQ, buy=0.03 (나머지 고정) Calmar | 0.300 | 실험 B |
| QQQ, ma=200 (나머지 고정) Calmar | 0.300 | 실험 C |

세 실험 모두에서 QQQ의 기대값이 0.300으로 동일한 이유: 현재 확정값(MA=200, buy=3%, sell=5%, hold=3)은 3개 실험 모두의 교집합이므로, 어느 실험에서든 해당 조합은 같은 백테스트를 실행하게 된다.

이 값들이 크게 다르면 고정 파라미터 설정에 오류가 있는 것이므로, 결과를 신뢰하기 전에 원인을 파악해야 한다. 소수점 둘째자리 수준의 미미한 차이는 라운딩에 의한 것이므로 허용한다.

### 10.2 결과 보고

실행이 완료되면 §7에서 정의한 형식(피벗 테이블 + 상세 CSV)으로 결과를 정리하여 보고한다. 결과에 대한 해석이나 판단은 포함하지 않아도 된다 (해석은 별도로 진행한다).
